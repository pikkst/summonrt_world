# T6.3 – Scan Ability for Boss Weakness Discovery

## Summary
Implemented the "Scan" ability for boss encounters in the SummonerWorld combat system. Players can now scan a boss to discover its elemental weaknesses and resistances, then strategically guess the primary weakness to amplify damage. A wrong guess applies a severe damage penalty.

## Implementation Details

### Scan Skill
- **Skill**: `scan` (non-damaging, costs 3 mana)
- Available to creatures as part of their skill pool
- Displays as "Scan" in combat skill grid

### Scan Flow
1. Player initiates `scanEnemy()` during their turn against a boss
2. System computes:
   - **Weaknesses**: Elements that have elemental advantage against the boss (e.g., `water` vs `fire` boss)
   - **Resistances**: Elements that have disadvantage against the boss
3. Results are logged and stored in `combat.scanResult`
4. Player sees a grid of weakness buttons to guess from

### Guess System
- After scanning, player clicks a weakness element to "guess" the primary target
- **Correct guess**: Damage amplified by +30%
- **Wrong guess**: Damage penalized by -70% (only 30% of normal damage)
- Guess result is logged and displayed in the UI
- State persists for the remainder of the combat

### Damage Calculation Update
Both `attackWithCreature` and `useSkill` now apply the guess multiplier:
```typescript
const guessMult = getGuessDamageMultiplier(combat.scanResult);
let playerDamage = Math.max(1, Math.floor(damageRaw * eff.factor * damageMult * affectionMult * guessMult + variance));
```

### UI Components
- **Boss Analysis Module**: Shown only when enemy is a boss and no scan has been performed yet
- **Scanned Intel Panel**: Displays weaknesses (green) and resistances (red) after scanning
- **Guess Grid**: Shows clickable weakness buttons; replaced by result indicator after guessing

## Helper Functions (Exported for Testing)
- `computeBossWeaknesses(elements)`: Returns all elements that have advantage against given boss elements
- `computeBossResistances(elements)`: Returns all elements that have disadvantage against given boss elements
- `getGuessDamageMultiplier(scanResult)`: Returns 1.3 for correct guess, 0.3 for wrong guess, 1.0 otherwise
