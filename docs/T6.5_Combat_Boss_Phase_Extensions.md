# T6.5 – Extend Completed Combat Phase Boss Mechanics

## Task Overview

T6.5 extends the already-completed T6.2 base boss mechanics by adding:

- Environmental hazard rotation (lava bursts, frost spikes, storm pulses)
- Summoner career bonus integration into boss phase calculations

## What Was Found

### Existing Implementation

T6.2 was already completed and provided the foundation:

- Boss phase thresholds at 75%, 50%, and 25% HP
- Random elemental shifts on phase change
- Random environmental hazard text messages (static strings without gameplay effect)
- Career combat bonuses (`damage_dealt_pct`, `damage_taken_pct`) already integrated into player attacks and enemy damage via `applyCombatCareerBonuses` and `applyDamageTakenReduction`

### Identified Gaps

1. **Hazards had no gameplay effect** – Environmental hazards were only log messages, not applying actual damage or status effects.
2. **No structured hazard data** – Hazards were plain strings with no element, damage value, or rotation logic.
3. **Career bonus bug** – `applyDamageTakenReduction` used the formula `1 - ((bonuses.damage_taken_pct || 0) / 100)`. For Summoner career nodes with `damage_taken_pct: -5`, this produced `1.05` (increasing damage instead of reducing it). The intent, consistent with other negative bonus values like `settlement_upgrade_cost_pct`, is for negative values to mean reduction. The formula was corrected to `1 + ((bonuses.damage_taken_pct || 0) / 100)`.

## What Was Done

### 1. Structured Boss Hazards (`src/types/game.ts`)

Added `BossHazard` interface to replace the plain string hazard:

```typescript
export interface BossHazard {
  key: string;
  name: string;
  element: Element;
  description: string;
  baseDamage: number;
}
```

Updated `BossPhase.hazard` and `CombatState.activeHazard` to use `BossHazard | undefined`.

### 2. Hazard Rotation Logic (`src/stores/game/modules/combatModule.ts`)

Replaced the random hazard array with structured hazard definitions:

| Hazard Key | Name | Element | Base Damage |
|------------|------|---------|-------------|
| `lava_burst` | Lava Burst | fire | 8 |
| `frost_spikes` | Frost Spikes | ice | 6 |
| `storm_pulse` | Storm Pulse | lightning | 7 |
| `poisonous_fumes` | Poisonous Fumes | nature | 5 |
| `shadow_miasma` | Shadow Miasma | darkness | 5 |
| `scorching_heat` | Scorching Heat | fire | 5 |

The rotation cycles deterministically by phase index:
- Phase 1 (75%) → `lava_burst`
- Phase 2 (50%) → `frost_spikes`
- Phase 3 (25%) → `storm_pulse`

### 3. Environmental Hazard Damage (`src/stores/game/modules/combatModule.ts`)

Added `applyEnvironmentalHazardDamage` action that:

- Checks if an active boss hazard exists
- Calculates damage: `baseDamage + random(0..2)`
- Applies Summoner career bonuses via `applyDamageTakenReduction`
- Updates the active creature's health
- Logs the hazard damage in the combat log
- Handles creature fainting from hazard damage (same logic as enemy attacks)

Hazard damage is triggered at the start of `triggerEnemyTurn` each round.

### 4. Summoner Career Bonus Integration

The existing Summoner path bonuses are now properly factored into boss phase calculations:

- **`damage_dealt_pct: 5`** (Creature Warden node) – Applied via `applyCombatCareerBonuses` during player attacks.
- **`damage_taken_pct: -5`** (Creature Warden node) – Applied via `applyDamageTakenReduction` for both enemy attacks and environmental hazard damage.

Fixed `applyDamageTakenReduction` formula in `src/data/careerTreeIntegration.ts`:

```typescript
// Before: 1 - ((bonuses.damage_taken_pct || 0) / 100)
// After:  1 + ((bonuses.damage_taken_pct || 0) / 100)
```

With `damage_taken_pct: -5`, the multiplier is `0.95`, providing the intended 5% damage reduction.

### 5. Type Safety (`src/stores/game/types.ts`)

Added `applyEnvironmentalHazardDamage: () => void` to the `GameActions` interface.

### 6. Tests (`src/test/bossPhaseMechanics.test.ts`)

Created new test file covering:

- Environmental hazard damage application during boss combat
- Hazard rotation across boss phases (lava_burst → frost_spikes → storm_pulse)
- Summoner career bonus reduction of hazard damage
- Summoner damage_dealt_pct increasing player damage
- Combined Summoner combat bonuses for boss phase calculations
