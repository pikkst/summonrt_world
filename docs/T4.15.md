# Task T4.15 - Add Creature Evolution Paths

## What was found
The codebase had `evolutionLevel` and `evolvesIntoKey` fields on `CreatureTemplate` and `evolutionStage` / `evolvedFromKey` on `CreatureInstance`, but they were never populated or consumed:

1. **`src/types/game.ts`**: Template and instance interfaces had evolution fields defined but unused
2. **`src/data/constants.ts`**: Had class-based `EVOLUTION_CHAINS` (common→uncommon→rare→etc.) used for stat-tier upgrades, but no species-specific evolution paths
3. **`src/modules/creatures/creatureFactory.ts`**: `getTemplateByKey()` was a stub returning `undefined`; no template registry existed
4. **`src/core/xpCurve.ts`**: `checkEvolution()` only evaluated class-based chains and applied generic stat multipliers, ignoring any template-defined evolution
5. **Encounters & spawning**: Creatures were purely random with no lineage, and starting creatures were absent on game init

## What was done

### 1. Added species-line definitions (`src/data/constants.ts`)
Created `SPECIES_LINES` map with 5 named evolution lines:
- `fang_line` (beast, fire/earth)
- `wyrm_line` (dragon, lightning/air)
- `wraith_line` (undead, darkness/ice)
- `golem_line` (construct, earth/iron)
- `spirit_line` (spirit, light/nature)

Each line defines 4 stages (common → uncommon → rare → epic) with explicit prefixes, suffixes, creature types, and `minEvolutionLevel` thresholds (10 / 25 / 45 / 70).

### 2. Built persistent template registry (`src/modules/creatures/creatureFactory.ts`)
- Added `registerTemplate()`, `getTemplateByKey()`, `getAvailableTemplatesForElement()`, and `clearTemplateRegistry()`
- Added `registerSpeciesLine()` to deterministically pre-generate all stages for a given line
- Added `pickRandomSpeciesKey()` and `getRandomSpeciesStage()` for encounter spawning
- `generateCreatureTemplate()` now accepts optional `speciesKey` / `speciesStage` parameters
- All generated templates are registered in the central `Map<string, CreatureTemplate>`
- Preserved original random template generation for bosses, wild encounters, and fallback spawns

### 3. Updated evolution logic (`src/core/xpCurve.ts`)
- `checkEvolution()` now checks the creature's current template for `evolvesIntoKey` first
- Template evolution uses the **current template's `evolutionLevel`** as the threshold
- Template evolution returns `EvolutionResult` with `templateKey`, `newClass`, and `evolvedFromKey`
- Falls back to the existing `EVOLUTION_CHAINS` class-based chain when no template evolution exists
- `applyCreatureXP()` now supports two evolution modes:
  - **Template evolution**: swaps `templateKey`, sets `evolvedFromKey`, updates `class`, `type`, `elements`, and `skills` from the evolved template
  - **Class evolution**: retains original stat-multiplier behavior
- Added `templateEvolutionKey` / `templateEvolutionFromKey` collectors to avoid forward-reference issues with `updatedCreature`

### 4. Wired species lines into gameplay
- **`src/core/gameEngine.ts`**: `generateEncounter()` now has a 35 % chance to spawn from a species line instead of pure random generation
- **`src/stores/game/modules/playerModule.ts`**: `initGame()` now assigns the player a deterministic starting creature from a randomly selected species line stage 0
- Starting creatures have `templateKey`, `type`, `elements`, and `evolutionStage` properly initialized

### 5. Added comprehensive tests (`src/test/evolution.test.ts`)
- Species-line generation: deterministic keys, correct classes, correct `evolvesIntoKey` chain, 5 lines × 4 stages = 20 distinct templates
- Template registry: `getTemplateByKey`, `getAvailableTemplatesForElement`
- Template evolution: level threshold gating, fallback to class-based chain
- `applyCreatureXP` integration: templateKey swap, `evolvedFromKey` persistence, element/skill inheritance from evolved template, single-evolution-per-batch rule
- Helper functions: `pickRandomSpeciesKey`, `getRandomSpeciesStage`

### 6. Fixed existing imports
- Re-exported `SKILL_TEMPLATES` and `TRAITS` from `creatureFactory.ts` so `combatModule.ts` and `gameStore.ts` imports continue to compile
