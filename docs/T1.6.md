# Task T1.6 - Implement canUnlockNode Function

## What was found
The career module (`careerModule.ts`) was using the old `MASTER_SKILL_TREE` system with `skillPoints` and `skillsUnlocked`. The new `nodes.json` career tree system with `unspent_passive_points` and `unlocked_node_ids` was already integrated in T1.4, but the core validation logic (`canUnlockNode`) was missing. Additionally, T1.7 (`getAggregateStats`) and T1.8 (`respecAllNodes`) were also missing.

## What was done

### 1. Implemented `canUnlockNode` function in `src/data/careerTree/index.ts`
Implements all 3 validation rules:
- **Rule A**: Checks `unspent_passive_points >= 1` - player must have at least one passive point to spend
- **Rule B**: Verifies target node is not already in `unlocked_node_ids` - prevents duplicate unlocks
- **Rule C**: Validates that at least one connection ID exists in `unlocked_node_ids` - ensures prerequisite chain

### 2. Implemented `getAggregateStats` function
Reads all unlocked node stats and aggregates them into a summed dictionary. Used for calculating total career bonuses.

### 3. Implemented `respecAllNodes` function  
Resets `unlocked_node_ids` back to `["root_hub"]` and refunds all spent points to `unspent_passive_points`.

### 4. Created test file `src/data/careerTree/careerTree.test.ts`
Tests all 3 rules for `canUnlockNode`:
- Rule A test: points = 0, expects false
- Rule B test: node already unlocked, expects false  
- Rule C test: no connected node unlocked, expects false
- Success test: all conditions met, expects true
- Non-existent node test: edge case handling

Tests for `getAggregateStats` and `respecAllNodes` verify correct stat summation and point refund behavior.
