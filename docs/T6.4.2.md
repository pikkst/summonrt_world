# Sprint 6 – T6.4.2: Add Multi-Path Guarantee

## Objective
Ensure that every generated dungeon floor contains at least 3 distinct shortest paths from the entrance room to the boss room. This is achieved by computing shortest paths using BFS and injecting controlled shortcut edges when fewer than 3 paths exist.

### Requirements

## What Was Done

### ensureMultipleShortestPaths (lines 302–354)
The function implements the following algorithm:

1. **Locate entrance and boss rooms** from the generated `DungeonFloorGraph`
2. **Compute current shortest paths** using `findAllShortestPaths(graph, startId, endId)`
3. **Early exit** if `paths.length >= 3`
4. **Build distance map** from boss room using `calculateRoomDistanceMap(graph, endId)`
5. **Collect all rooms on existing shortest paths** into a `pathRooms` set
6. **Iterate through off-path rooms**:
   - Skip rooms already on initial shortest paths
   - Find rooms at `bossDist + 1` and `bossDist - 1` with available connection slots
   - Add bidirectional edges connecting the current room to both distance layers
   - **Re-validate** by calling `findAllShortestPaths()` after each edge addition
   - Break out of loop once `paths.length >= 3`

### BFS Pathfinding Utilities
- **findShortestPath**: Standard BFS returning the first shortest path (or null if unreachable)
- **findAllShortestPaths**: BFS variant tracking `shortestLength` and collecting all paths of exactly that length
- **calculateRoomDistanceMap**: Single-source BFS producing a `Map<string, number>` of distances from a start room
- **isOnShortestPath**: Helper that checks whether a room lies on any shortest path by verifying it has a neighbor one step closer to the boss

## Algorithm Details

### Edge Selection Strategy
Instead of adding purely random edges, the algorithm selects rooms based on their distance to the boss:
- A room at distance `d` from the boss can connect to rooms at `d+1` (closer to boss) and `d-1` (further from boss)
- This creates "shortcut" corridors that bypass segments of the original longest single path
- The result is that multiple equally-short routes emerge without breaking maze connectivity

### Path Uniqueness Validation
After each pair of edges is added, `findAllShortestPaths()` is called again. If the count reaches 3, the function stops. This ensures:
- We never over-modify the graph
- Deterministic behavior (same seed = same edges)
- Shortest path lengths do not increase
