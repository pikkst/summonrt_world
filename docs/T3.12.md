# T3.12 - Standardize Encounter XP Formula

## What was found
The `calculateEncounterXP` function was already implemented in `src/core/xpCurve.ts` with the correct formula: `(MonsterBaseXP × MonsterLevel) × WorldMod × AffinityBonus`. However, it was not being used consistently throughout the codebase. Two critical combat paths were using different XP calculation methods:

1. **`src/stores/game/modules/combatModule.ts`** - `handleVictory` used raw `baseExpValue` for creature XP and `baseExpValue / 4 × WorldMod` for player XP, without applying monster level or affinity bonuses.
2. **`src/core/missionQueue.ts`** - `resolveAutomatedCombat` used a hardcoded `20 × xpFactor` for all XP gains, ignoring monster level, world modifier, and elemental affinity.

This inconsistency meant that the encounter XP formula was only defined in tests but not enforced in actual gameplay logic.

## What was done
- **Standardized creature XP in combatModule.ts**: Replaced direct `baseExpValue` usage with `calculateEncounterXP(baseExpValue, currentWorldId, getWorldModifier(currentWorldId), player.affinity.primary, enemyTemplate?.elements)`. Monster level is now derived from the player's `currentWorldId` (matching the encounter generation tier), and elemental affinity is computed from the player's primary element vs the enemy's elements.
- **Standardized player XP in combatModule.ts**: Player encounter XP now follows the same formula, divided by 4, ensuring consistent scaling across all combat rewards.
- **Standardized automated combat XP in missionQueue.ts**: Updated `resolveAutomatedCombat` to accept optional `worldLayer` and `atkElements` parameters. When provided, XP for each defeated enemy is calculated via `calculateEncounterXP` using the target's `baseExpValue` and `level`, plus the appropriate world modifier and elemental affinity. Backward compatibility is preserved for existing callers.
- **Fixed missing import**: Added the missing `SeededRandom` import in `combatModule.ts` to resolve pre-existing TypeScript errors.
- **Cleaned dead code**: Removed the unused `xpFactor` option from `resolveAutomatedCombat` and unused `getAffinityBonusXP` import from `combatModule.ts`.
