# Task T4.1 - Update Element Constants

## What was found
The codebase the elemental system had an inconsistency where quest-only elements were appearing in starter element roll pools:

1. **`src/stores/game/helpers.ts`**: `rollAffinity()` function used all 13 elements (including void/starlight/chaos) for player affinity rolls
2. **`src/modules/creatures/creatureFactory.ts`**: Had a duplicate local ELEMENTS constant instead of importing from the central constants file
3. **`src/types/game.ts`**: Element type included all 13 elements (fire through chaos)
4. **`src/core/xpCurve.ts`**: ELEMENTAL_OPPOSITIONS containing void/starlight/chaos relationships for XP bonus calculations

## What was done
1. **Updated `src/data/constants.ts`**:
   - `ELEMENTS` now contains exactly 10 starter elements: fire, water, earth, air, lightning, iron, nature, ice, light, darkness
   - Added `ALL_ELEMENTS` containing all 13 elements including void, starlight, chaos for quest-only unlock mechanics
   - Added `AllElement` type export for type safety when working with all elements

2. **Updated `src/data/index.ts`**:
   - Added export for `ALL_ELEMENTS` and `AllElement` type

3. **Updated `src/stores/game/helpers.ts`**:
   - `rollAffinity()` now uses only the 10 starter elements for player affinity generation
   - void/starlight/chaos can only be obtained through quest rewards via the `learned` array in ElementalAffinity

4. **Updated `src/modules/creatures/creatureFactory.ts`**:
   - Removed duplicate local ELEMENTS constant
   - Now imports ELEMENTS from `../../data/constants.ts` for creature template generation
   - Creature elements also constrained to the 10 starter elements

5. **Preserved quest-only element support**:
   - The `Element` type in `src/types/game.ts` still includes all 13 elements
   - Quest reward system in `completeQuest()` can still grant void/starlight/chaos through `affinity.learned` (lines 501-507)
   - Fusion system can still produce void/starlight elements as mutations (Lines 567-575 in missionModule.ts)
   - `xpCurve.ts` retains void/starlight/chaos in ELEMENTAL_OPPOSITIONS for proper XP bonus calculations

## Summary
The elemental system now distinguishes between:
- **Starter Elements (10)**: Available through random rolls during character creation and creature generation
- **Quest-Only Elements (3)**: void, starlight, chaos - accessible only through quest completion and special fusion events

This aligns with the GDD requirement that these three elements are "quest-only unlocks" and should not appear in the standard starter pool.
