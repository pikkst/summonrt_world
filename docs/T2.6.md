# Task T2.6 – Write Timer Compression Formula

## What was found

The mission system was using raw `duration_seconds` values without any speed modifiers. The `createActiveMission` factory in `src/core/missionQueue.ts` (line 45) computed `end_time` as `now + duration_seconds * 1000` and stored the uncompressed duration in the mission record.

Career tree stats (e.g. `exploration_speed_pct`, `caravan_speed_pct`) were already defined in `src/data/careerTree/nodes.json` and aggregated by `getAggregateStats()` in `src/data/careerTree/index.ts`, but these values were not connected to mission timing. Creature speed (`baseSpeed` / `speed`) existed on `CreatureTemplate` and `CreatureInstance`, but it was also unused for mission duration.

The `MissionModifiers` interface contained only `speed_multiplier`, `caravan_speed_pct`, and `yield_bonus_pct` — there were no typed fields for tree speed or creature agility, so any compression logic would have required ad-hoc string keys.

## What was done

### 1. Added `calculateCompressedDuration` to `missionQueue.ts`

The core formula is now a pure, reusable function in `src/core/missionQueue.ts`:

```typescript
export function calculateCompressedDuration(
  baseDuration: number,
  treeSpeedMod: number,
  creatureAgilityMod: number
): number {
  const totalModifier = treeSpeedMod + creatureAgilityMod;
  const reductionFraction = Math.min(0.9, totalModifier / 100);
  return Math.max(60, baseDuration * (1 - reductionFraction));
}
```

Behavior:
- `treeSpeedMod` and `creatureAgilityMod` are summed to a total percentage reduction.
- Reduction is capped at **90%** so missions never instant-complete.
- Minimum final duration is **60 seconds** to prevent degenerate short timers.

### 2. Added `getCreatureAgilityMod` helper

```typescript
export function getCreatureAgilityMod(creatures: CreatureInstance[]): number {
  if (creatures.length === 0) return 0;
  const avgSpeed = creatures.reduce((sum, c) => sum + (c.speed || 0), 0) / creatures.length;
  return Math.round(avgSpeed);
}
```

This converts assigned creature `speed` stats into a percentage modifier. A party of creatures with average speed 12 contributes a 12% reduction.

### 3. Extended `MissionModifiers` interface

Added two typed fields:
- `tree_speed_pct?: number` — total career tree speed stat (from `getAggregateStats`)
- `creature_agility_mod?: number` — derived from assigned creatures via `getCreatureAgilityMod`

### 4. Updated `createActiveMission` to apply compression

`createActiveMission` now accepts an optional `modifiers` object. When `tree_speed_pct` or `creature_agility_mod` are present, the factory computes the compressed `duration_seconds` and `end_time` via `calculateCompressedDuration`. Other modifiers (`yield_bonus_pct`, `caravan_speed_pct`, etc.) are preserved in the mission record.

### 5. Added tests for all new logic

`src/test/missionQueue.test.ts` — 15 test cases covering:
- Zero-modifier passthrough
- Individual tree and creature modifiers
- Combined modifiers
- 90% reduction cap
- 60-second minimum floor
- Fractional modifier accuracy
- Empty creature array, single creature, multiple creatures, and missing speed defaults
