## What was found

The existing `src/test/missionQueue.test.ts` contained unit tests for individual functions (`calculateCompressedDuration`, `getCreatureAgilityMod`, `createActiveMission`), but lacked the required property-based testing with 1000 random missions. The mission queue system had already been implemented in `src/core/missionQueue.ts` with:

- **MissionType** union: `EXPLORE_TIER_1`, `SCOUT_DUNGEON`, `SMELT_ORE`, `CRAFT_ITEM`, `STORE_VISIT`, `TAX_EDICT`, `CARAVAN_ROUTE`
- **MissionStatus** union: `PENDING`, `IN_PROGRESS`, `COMPLETED`, `FAILED`, `CANCELLED`
- **ActiveMission** schema with all required fields
- Timer compression formula with 60-second minimum duration

The heartbeat system in `src/core/heartbeat.ts` already handled offline catch-up and batch resolution of expired missions.

## What was done

### Added Property-Based Tests to `src/test/missionQueue.test.ts`

Created a comprehensive test suite with 6 new test cases covering 1000 random missions each:

1. **`invariant: end_time >= start_time for all generated missions`**
   - Validates that `end_time` is always greater than or equal to `start_time`
   - Uses deterministic mission generation across all 7 mission types

2. **`invariant: IN_PROGRESS missions resolve to COMPLETED or FAILED when expired`**
   - Simulates expiration and verifies valid status transitions
   - Confirms the status field correctly transitions from `IN_PROGRESS` to terminal states

3. **`invariant: mission duration never below minimum threshold (60s)`**
   - Verifies the timer compression formula respects the 60-second floor
   - Ensures `calculateCompressedDuration` correctly applies the minimum

4. **`invariant: modifiers do not exceed valid ranges`**
   - Validates `tree_speed_pct` stays within 0-50 range
   - Validates `creature_agility_mod` stays within 0-30 range

5. **`invariant: world_layer stays within valid game bounds (1-100)`**
   - Confirms all generated missions have valid world_layer values

6. **`property: 1000 random missions with end_time in past resolve correctly via heartbeat logic`**
   - Simulates the heartbeat's batch resolution behavior
   - Verifies that expired `IN_PROGRESS` missions are correctly identified for processing

### Additional Unit Tests Added

- `isMissionExpired`: Tests for future/past/expired states
- `getRemainingSeconds`: Tests countdown calculation and expiration handling
- `MissionStatus transitions`: Validates status value constraints and transition possibilities
