# Task T3.8 - Fix `getLevelThreshold()` to use GDD formula

## What was found
The `GameEngine` class in `src/core/gameEngine.ts` contained a legacy `getLevelThreshold(level: number): number` method that duplicated the XP threshold formula already implemented in `src/core/xpCurve.ts` as `getXPThreshold(level: number): bigint`.

Key issues with the legacy method:
- It was dead code — no callers existed anywhere in the codebase.
- It returned `number` instead of `bigint`, risking precision loss for high levels (Level 1000+).
- It used `Math.floor()` rounding, while the canonical `getXPThreshold` uses `Math.round()` + BigInt, matching the GDD formula exactly.
- The mission completion and creature tick systems already used the correct `getXPThreshold` from `xpCurve.ts`.

## What was done
- Removed the legacy `GameEngine.getLevelThreshold()` method from `summoner-world/src/core/gameEngine.ts`.
- Confirmed `tickCreatures()` and `applyCreatureXP()` both use the canonical `getXPThreshold()` from `xpCurve.ts`, ensuring a single source of truth for XP curves.
- No other callers existed, so no call-site updates were required.

## GDD formula reference
From `SummonerWorld_GDD.md` §5.1:
- `XP_Base = 100 × (1.15)^(Level - 1)`

This is now enforced exclusively through `src/core/xpCurve.ts`.
