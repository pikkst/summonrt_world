# T1.4 - Career Tree PlayerState Extension

## Task Description
Extend `PlayerState` with career tree fields: `unspent_passive_points` and `unlocked_node_ids`.

## Implementation Details

### Changes Made

#### 1. Extended `PlayerState` type in `src/types/game.ts`
Added two new fields to the `PlayerState` interface:
- `unspent_passive_points: number` - Tracks available passive points for unlocking career nodes
- `unlocked_node_ids: string[]` - Array of unlocked career node IDs

#### 2. Updated `playerModule.ts` - Character Creation
Modified the `initGame` function to initialize both fields:
- `unspent_passive_points: 0` - Starts at zero, earned through gameplay
- `unlocked_node_ids: ['root_hub']` - Root hub node is unlocked by default for all characters

#### 3. Updated `playerModule.ts` - Server Sync (register, login, syncWithServer)
Added compatibility for the new fields in:
- `register` function: `unspent_passive_points ?? 0`, `unlocked_node_ids ?? ['root_hub']`
- `login` function: `unspent_passive_points ?? 0`, `unlocked_node_ids ?? ['root_hub']`
- `syncWithServer` function: `unspent_passive_points ?? 0`, `unlocked_node_ids ?? ['root_hub']`

#### 4. Updated `playerModule.ts` - Local Save/Load
Modified `loadGame` to restore or default the career tree fields from saved game data.

## Files Modified
- `src/types/game.ts` - Added `unspent_passive_points` and `unlocked_node_ids` to `PlayerState`
- `src/stores/game/modules/playerModule.ts` - Initialized and restored career tree fields

## Status
**COMPLETED** - PlayerState now supports the career tree system foundation.
