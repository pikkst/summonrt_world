# Task T2.1 – Create missionQueue.ts with ActiveMission Schema

## What was found

The `src/core/` directory contained only `gameEngine.ts` and `worldGenerator.ts`. There was no mission queue module or active mission schema. The existing `missionModule.ts` in `src/stores/game/modules/` handled only immediate player actions (movement, exploration, combat) using `setTimeout` callbacks, but the time-based mission system required by Sprint 2 was missing entirely.

No `MissionType`, `MissionStatus`, or `ActiveMission` types existed in the codebase. The project uses string-literal union types for categorical enums (e.g., `QuestStatus`, `CombatPhase`) rather than TypeScript `enum` declarations, so the new types follow that convention.

## What was done

### 1. Created `src/core/missionQueue.ts`
Introduced the foundational types and helpers for the time-based execution engine:

- **`MissionType`** – string-literal union covering the 7 mission kinds required by the GDD: `EXPLORE_TIER_1`, `SCOUT_DUNGEON`, `SMELT_ORE`, `CRAFT_ITEM`, `STORE_VISIT`, `TAX_EDICT`, `CARAVAN_ROUTE`
- **`MissionStatus`** – lifecycle states: `PENDING`, `IN_PROGRESS`, `COMPLETED`, `FAILED`, `CANCELLED`
- **`MissionModifiers`** – open-ended key-value map for mission-specific stat adjustments (`speed_multiplier`, `caravan_speed_pct`, `yield_bonus_pct`, etc.)
- **`ActiveMission`** – core schema with fields: `mission_id`, `type`, `assigned_creatures[]`, `world_layer`, `start_time`, `duration_seconds`, `end_time`, `status`, `modifiers{}`
- **Factory helper `createActiveMission`** – generates a new mission with a unique ID, sets `start_time` to `Date.now()`, computes `end_time`, and initializes status to `PENDING`
- **Time utility helpers**:
  - `isMissionExpired(mission, currentTime?)` – checks if `currentTime >= end_time`
  - `getRemainingSeconds(mission, currentTime?)` – returns seconds until expiry, clamped to 0

### 2. Verified type compatibility
The new module imports no store dependencies, keeping it decoupled from Zustand so it can be reused by both the client and future server-side execution engine.
