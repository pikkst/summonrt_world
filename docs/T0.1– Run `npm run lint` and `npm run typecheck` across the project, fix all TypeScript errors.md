# Sprint 0 – Task T0.1 Documentation

## Objective
Run project-wide TypeScript type checking (`tsc --noEmit`) and resolve all type errors to ensure the prototype compiles cleanly.

## Findings

### Environment
- The project root did **not** contain `lint` or `typecheck` npm scripts.
- `package.json` only had: `dev`, `build` (`tsc && vite build`), and `preview`.
- No ESLint/Prettier configs were present.

### Locating the Error
Running `npx tsc --noEmit --project tsconfig.json` revealed **1 TypeScript error**:

```
src/stores/gameStore.ts(194,19): error TS2322
Type '{ player: any; ...' is not assignable to type 'GameStore'.
... is missing the following properties from type 'GameActions':
logout, openCommunity, openProfile, refreshCommunity, and 20 more.
```

### Root Cause
The Zustand store in `summoner-world/src/stores/gameStore.ts` defines a `GameActions` interface with **24 methods** but the store object only implemented a subset of them. The remaining 24 methods were type-declared in the interface but never given function bodies in the store creation call (`create<GameStore>()(subscribeWithSelector(...))`).

Missing methods:
- `logout`
- `openCommunity`
- `openProfile`
- `refreshCommunity`
- `getCommunityPlayers`
- `selectCommunityPlayer`
- `sendCommunityMessage`
- `loadCommunityMessages`
- `loadCommunityConversations`
- `blockUser`
- `unblockUser`
- `reportUser`
- `createTrade`
- `acceptTrade`
- `declineTrade`
- `invitePlayerToParty`
- `acceptPartyInvite`
- `declinePartyInvite`
- `leaveParty`
- `createGuild`
- `invitePlayerToGuild`
- `acceptGuildInvite`
- `declineGuildInvite`
- `leaveGuild`

## Changes Made

### 1. Added `typecheck` script to `summoner-world/package.json`
```json
"typecheck": "tsc --noEmit"
```

### 2. Implemented stub actions in `src/stores/gameStore.ts`
Added 24 missing action methods to the Zustand store object. Each stub:
- Checks state prerequisites (`get().player`) when required.
- Uses existing API base (`http://localhost:5000/api`) for network calls.
- Accesses `appendLog` via `const { appendLog } = get()` to follow store conventions.
- Provides `console.error` fallbacks for failed requests.
- Returns appropriate UI state changes where applicable (`openCommunity`, `openProfile`, `logout`).

Example stubs added:
```typescript
logout: () => {
  set({ player: null, initialized: false, screen: 'login' });
},
openCommunity: (tab?: CommunityState['tab']) => {
  const { community } = get();
  set({ screen: 'community', community: { ...community, tab: tab || 'nearby' } });
},
createGuild: async (name: string) => { /* ... */ },
acceptTrade: async (tradeId: string) => { /* ... */ },
// ... 24 total
```

## Result
- `tsc --noEmit` now exits with **0 errors**.
- The store object is fully assignable to the `GameStore` intersection type.
- Future sprints (MMO/Online Infrastructure) can now safely flesh out these network-backed actions without additional structural type fixes.
