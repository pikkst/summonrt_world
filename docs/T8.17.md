# T8.17 – Profession System

## Sprint 8 Task: T8.17

### Objective

Add player-owned profession progression for the six Sprint 8 profession paths:

- Blacksmith
- Explorer
- Shopkeeper
- Broker
- Official
- Summoner

The system is stored under `PlayerCoreState` so profession growth follows the player root aggregate and can be serialized with existing Player Core save/load.

### Changes Made

#### 1. New module `src/core/playerCore/professionCore.ts`

The profession module defines:

```text
ProfessionId
ProfessionProgression
ProfessionState
ProfessionDefinition
ProfessionPerkDefinition
```

Each profession starts at level 1 with 0 XP and tracks:

- current level
- current level XP
- total XP earned
- unlocked perk IDs
- last advancement timestamp

#### 2. Six profession progression tracks

| Profession | XP Sources | Early Bonuses |
|------------|------------|---------------|
| Blacksmith | smelting, item crafting | `smelting_speed_pct`, `crafting_speed_pct`, `material_retention_chance` |
| Explorer | tile exploration, secret discovery | `exploration_speed_pct`, `map_reveal_radius`, `secret_discovery_chance` |
| Shopkeeper | store sales, bulk sales | `store_traffic_pct`, `selling_price_pct`, `rare_item_sourcing_chance` |
| Broker | trade routes, arbitrage | `caravan_speed_pct`, `tariff_discount_pct`, `arbitrage_opportunity_chance` |
| Official | tax collection, policies | `tax_revenue_pct`, `public_order_pct`, `settlement_upgrade_cost_pct` |
| Summoner | creature contracts, summoning | `capture_bonus_pct`, `creature_affection_gain_pct`, `creature_xp_gain_pct` |

Perks unlock deterministically at levels 2, 5, and 10.

#### 3. Deterministic profession XP curve

Profession XP uses named constants:

```text
PROFESSION_BASE_XP = 100
PROFESSION_XP_GROWTH = 1.18
PROFESSION_LEVEL_CAP = 100
```

`addProfessionXp()` applies integer XP, handles multi-level gains, caps at level 100, and refreshes unlocked perks from definitions.

#### 4. Player Core and save/load integration

`PlayerCoreState` now includes optional `professions` for backward compatibility with old fixtures and saves.

Default creation and migration populate:

```text
professions: createDefaultProfessionState()
```

Save deserialization normalizes missing or invalid profession state with safe defaults through `normalizeProfessionState()`.

#### 5. Profession aggregate bonuses

`getProfessionAggregateBonuses()` sums unlocked profession perk bonuses into the same stat-key style used by career tree integration, so later economy, exploration, combat, and summoning systems can consume one plain bonus map.

### Tests Added

- Explorer, Shopkeeper, Broker, Official, and Summoner progression bonuses
