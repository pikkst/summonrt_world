# T6.0.6 – Player Defeat Transition to Demonlord Status

## Task
Add player defeat transition to Demonlord status in `combatModule.ts`.

## Findings

The combat defeat branch in `src/stores/game/modules/combatModule.ts` (`triggerEnemyTurn`) previously handled total party wipe unconditionally: it set the combat phase to `'defeat'` and deducted 15 Life points. There was no consideration for whether the player was on a Demonlord floor (floors 50–100, defined by `DEMONLORD_FLOOR_MIN = 50` and `DEMONLORD_FLOOR_MAX = 100` in `src/core/demonlord.ts`).

The `DemonlordState` type and `createDemonlordState()` factory already existed, and `demonlordModule.ts` provided all supporting challenge/accept/resolve functions. The only missing piece was the trigger condition in combat: when a player is defeated with all summons fainted on a Demonlord floor and no current lord yet assigned, the player should claim the Demonlord title instead of losing Life.

## Changes Made

### `src/stores/game/modules/combatModule.ts`

**1. Added imports:**
```typescript
import { createDemonlordState, getDemonlordForFloor } from '../../../core/demonlord';
```

**2. Rewrote the final defeat branch** (all summons fainted) in `triggerEnemyTurn`:

The previous single-path logic:
```typescript
set((s: any) => ({
  combat: { ...s.combat, log: enemyLog, phase: 'defeat' },
  player: {
    ...s.player,
    life: { ...s.player.life, current: Math.max(1, s.player.life.current - 15) },
    creatures: s.player.creatures.map(...),
  },
}));
```

was replaced with a conditional that checks two things before granting the title:
- `onDemonlordFloor` — `getDemonlordForFloor(currentWorldId)` returns true (floor 50–100)
- `!alreadyLord` — no current lord is assigned (`demonlordState.currentLordPlayerId !== player.id`)

**Demonlord claim path** (floor 50–100, no current lord):
```typescript
const newDemonlordState = createDemonlordState();
newDemonlordState.currentLordPlayerId = player.id;
newDemonlordState.currentLordPlayerName = player.name;
newDemonlordState.influence = 100;
```
The combat state is set to `'defeat'` and `enemyHp: 0`, creatures are updated, and `demonlordState` is stored. No Life penalty is applied. A system log message is appended:
> 🔥 You have been defeated, but your will burned brighter than the flames! You have claimed the Demonlord title for Floor {floor}!

**Normal defeat path** (all other cases):
- If already the Demonlord: a special message is shown (title preserved, try again). No Life penalty.
- All other cases: standard −15 Life penalty is applied.
