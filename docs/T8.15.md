# T8.15 – Integrate Career Passives into Economy

## Sprint 8 Task: T8.15

### Objective

Connect the Shopkeeper, Broker, and Official career-tree passives to the economy simulation so player progression actually shifts the numbers produced by `housingEconomy`, `merchantLogic`, `tradeCaravan`, and mission-duration calculation.

### Career Passives Wired

| Career | Stat Key | Effect | Wired Into |
|--------|----------|--------|------------|
| Shopkeeper | `selling_price_pct` | Increases the maximum unit price a merchant will pay the player for a good | `merchantLogic.merchantBuyFromPlayer` via `applySellingPriceBonus` |
| Shopkeeper | `store_traffic_pct` | Speeds up STORE_VISIT and other shop-related missions | `applyMissionSpeedBonuses` (max with caravan) |
| Broker | `caravan_speed_pct` | Speeds up CARAVAN_ROUTE missions | `applyMissionSpeedBonuses` (max with traffic) |
| Broker | `tariff_discount_pct` | Reduces the caravan tariff percentage applied on arrival | `tradeCaravan.resolveCaravanTrade` via `applyCaravanTariffDiscount` |
| Official | `tax_revenue_pct` (settlement_tax_revenue_pct) | Boosts the housing / settlement tax yield | `housingEconomy.processHousingEconomyTick` via `adjustHousingTaxRate` |

### Changes Made

#### 1. New module `src/core/economy/careerEconomy.ts`

A pure helper module that contains every economy-side adapter for career bonuses. Keeping these helpers centralized avoids duplicating ceiling/floor logic in callers.

```text
applySellingPriceBonus(basePrice, bonuses)
applyCaravanTariffDiscount(baseTaxRatePct, bonuses)
applySettlementTaxRevenueBonus(baseTaxAmount, bonuses)
adjustHousingTaxRate(baseTaxRatePct, bonuses)
applyMissionSpeedBonuses(baseDurationSeconds, bonuses)
```

Caps are exposed as named constants:

- `MAX_SELLING_PRICE_BONUS_PCT = 50`
- `MAX_TARIFF_DISCOUNT_PCT = 75`
- `MAX_TAX_REVENUE_BONUS_PCT = 100`
- `MAX_HOUSING_TAX_RATE_PCT = 50`
- Mission duration reduction cap mirrors `calculateCompressedDuration` (90% / 60s floor).

#### 2. Merchant buy acceptance now respects Shopkeeper `selling_price_pct`

`merchantBuyFromPlayer` previously rejected any `unitPrice > entry.sellPrice`. The ceiling is now `applySellingPriceBonus(entry.sellPrice, careerBonuses)`, so a Shopkeeper passive lifts the merchant's willingness to pay the player without breaking the existing undercut mechanic (which still keys off `entry.buyPrice`).

#### 3. Caravan tariff now respects Broker `tariff_discount_pct`

`resolveCaravanTrade` accepts an optional `careerBonuses` argument, returns the effective `taxRatePct`, and applies the discounted rate to the profit share calculation. The return shape was extended (backwards compatible — existing fields unchanged) to include `taxRatePct` so the mission log can surface the reduction to the player.

#### 4. Housing tax now respects Official `tax_revenue_pct`

`processHousingEconomyTick` accepts an optional `careerBonuses` argument and uses `adjustHousingTaxRate` to scale `HOUSING_TAX_RATE_PCT` before computing the housing tax. This keeps the existing baseline tax rate for players with no Official passives.

#### 5. Mission duration now uses `caravan_speed_pct` / `store_traffic_pct`

`applyMissionSpeedBonuses` is the largest-of-two helper that compresses a base duration using the better of the two passives. It is wired into the `dispatchCaravan` flow in `missionModule.ts` so newly created CARAVAN_ROUTE missions already account for the Broker's caravan speed. The same helper is available for the future STORE_VISIT path implementation (T8.16.1).

#### 6. Wiring from store to economy

`missionModule.ts` was updated to:

- Fetch aggregated career stats in the housing economy tick callback and pass them to `processHousingEconomyTick`.
- Fetch aggregated career stats in `resolveCaravanMission` and pass them to `resolveCaravanTrade`.
- Fetch aggregated career stats in `dispatchCaravan` and pass them to `applyMissionSpeedBonuses`.
- Surface the reduced tariff in the arrival log when the effective rate is below the base 10%.
