# Task T2.11 – Offline Catch-Up Testing with Reward Accumulation

## What was found

The mission queue system and heartbeat module already had offline catch-up capabilities implemented:

- **`src/core/heartbeat.ts`**: The `createHeartbeat` function supports offline catch-up by checking `currentTime >= mission.endTime` and resolving expired missions via `resolveMission()`
- **`src/stores/game/modules/missionModule.ts`**: The `processOfflineCatchUp` method at line 935 already handles batch resolution of expired missions on login
- **`src/test/heartbeat.test.ts`**: Already had basic offline tests (lines 414-553) but lacked comprehensive reward accumulation verification

The existing tests verified:
- Mission resolution after 8h offline
- PENDING and COMPLETED mission handling during catch-up
- Basic heartbeat tick behavior

However, the tests did not verify reward accumulation across multiple mission types during extended offline periods.

## What was done

### Added 5 comprehensive offline catch-up tests to `src/test/heartbeat.test.ts`

#### Test 1: "should resolve 8h worth of missions and accumulate rewards correctly"
- Creates 7 missions with varied durations (10s to 4h) and 1 future mission (10h)
- Simulates 8-hour offline period with login at `logoutTimestamp + 8h`
- Verifies exactly 7 missions are resolved (future mission preserved)
- Confirms reward accumulation across all resolved mission types:
  - EXPLORE_TIER_1: exploration_loot + 50 XP
  - SMELT_ORE: smelted_ore + 25 XP
  - CRAFT_ITEM: crafted_item + 75 XP
  - STORE_VISIT: gold + 30 XP
  - CARAVAN_ROUTE: trade_goods + 100 XP

#### Test 2: "should correctly calculate mission duration compression during offline catch-up"
- Verifies that mission modifiers (tree_speed_pct, creature_agility_mod) persist through offline resolution
- Ensures compressed duration calculations are preserved correctly

#### Test 3: "should handle multiple combat missions and aggregate XP correctly"
- Tests SCOUT_DUNGEON mission type with multiple assigned creatures
- Validates XP is aggregated based on creature count (3 creatures = 90 XP)
- Confirms reward stacking with creature-specific rewards

#### Test 4: "should preserve future missions during offline catch-up"
- Verifies IN_PROGRESS missions with end_time beyond login timestamp remain in queue
- Ensures expired missions are resolved while future missions are unharmed

#### Test 5: "should accumulate total rewards across 8h with consistent aggregation"
- Generates 50 missions spread evenly across the 8-hour window
- Verifies all 50 missions resolve during catch-up
- Confirms all 7 mission types receive their respective rewards
- Validates consistent XP and reward counter increments
