# Task T2.8 – Implement Automated Combat Engine

## What was found

The codebase had an existing interactive combat system in `src/stores/game/modules/combatModule.ts` and a simpler `runCombat` function in `src/modules/combat/combatSystem.ts`, but no automated combat engine that could run simulations without UI interaction. 

- `combatModule.ts` handles turn-by-turn player interaction with `startCombat`, `attackWithCreature`, and `handleVictory` functions
- `combatSystem.ts` had a basic `runCombat` that only handled single attacker/defender pairs
- Mission queue (`src/core/missionQueue.ts`) needed integration with an automated combat system for time-based missions

## What was done

### 1. Added `resolveAutomatedCombat` function to `src/core/missionQueue.ts`

The automated combat engine is now a pure function that takes two teams of creatures and returns a `MissionResult`:

```typescript
export function resolveAutomatedCombat(
  teamA: CreatureInstance[],
  teamB: CreatureInstance[],
  options?: { rngSeed?: number; xpFactor?: number }
): MissionResult
```

**Features:**
- **30-turn maximum loop** — Combat always terminates within 30 turns
- **Turn-by-turn log generation** — Detailed combat log with turn markers and damage reports
- **Victory/defeat detection** — Team victory determined by surviving creatures
- **Elemental effectiveness** — Uses advantage/disadvantage system (1.5x super effective, 0.5x not very effective)
- **Experience rewards** — XP calculated from defeated enemy `baseExpValue`
- **Optional SeededRandom** — Deterministic combat for testing via `rngSeed` option
- **Optional XP factor** — Multiplies XP rewards via `xpFactor` option

### 3. Added `CombatTeamMember` interface

Helper interface for tracking creature state during simulation:

```typescript
export interface CombatTeamMember {
  creature: CreatureInstance;
  isAlive: boolean;
}
```

### 4. Added `baseExpValue` to `CreatureInstance` type

Extended `src/types/game.ts` to include optional `baseExpValue` field on creature instances for XP calculation during automated combat.

### 5. Created comprehensive test suite

`src/test/combat.test.ts` — 8 test cases covering:

- 30-turn maximum cap enforcement
- Battle log structure verification
- Elemental effectiveness application
