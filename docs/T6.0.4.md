# T6.0.4 – Demonlord Store Module Implementation

## Summary
Created `src/stores/game/modules/demonlordModule.ts` as a Zustand module providing reactive store actions for managing the Demonlord state within the game store. This module integrates the core Demonlord AI logic with the store layer, enabling player interaction with the Demonlord challenge system.

## What Was Done

### 1. Created `src/stores/game/modules/demonlordModule.ts`

The module exports `demonlordActions` function that receives Zustand's `set` and `get` functions and returns an object with the following actions:

#### `initDemonlord()`
Initializes the Demonlord state in the store using `createDemonlordState()` from the core module. Only initializes if no state already exists.

#### `challengeDemonlord()`
Issues a challenge to the Demonlord. Validates:
- Player must be logged in
- Player must be on floors 50+ (Demonlord floors)
- Adds player to pending challenges queue

#### `acceptDemonlordChallenge()`
Accepts the first pending challenge, moving it to active challenge state. Used for Demonlord AI to accept player challenges.

#### `resolveDemonlordCombat(victorId: string)`
Resolves a Demonlord combat encounter:
- Player victory: Sets challenger as new Demonlord, awards +5 passive points
- Demonlord victory: Clears active challenge, maintains current Demonlord

#### `isOnDemonlordFloor(): boolean`
Helper that checks if the current world ID is on floors 50+ where the Demonlord can be challenged.

#### `updateDemonlordInfluence(playerCountDelta: number)`
Updates Demonlord influence based on player activity:
- Modifies floor activity tracking
- Recalculates influence based on player count and activity
- Updates store state with new influence value

#### `getDemonlordInfluenceForCombat(): number`
Returns the current influence value for combat modifiers.

#### `applyDemonlordCombatBonuses(baseDamage, baseDefense)`
Calculates combat bonuses based on:
- Demonlord influence modifiers (damage dealt +50% at full influence, damage taken -30% at full influence)
- Career tree bonuses (damage_dealt_pct, damage_taken_pct)

#### `setDemonlordFloorActivity(activity)` / `getDemonlordFloorActivity()`
Getters/setters for floor-specific activity tracking.

## Design Rationale

### Store Integration Pattern
The demonlordModule follows the same pattern as other modules (playerModule, combatModule, careerModule):
- Pure actions object that receives `set` and `get`
- Integrates with existing career tree and bonus systems
- Uses core demonlord functions for business logic

### Floor Activity Model
The `FloorActivity` interface tracks:
- `playerCount`: Current players on the floor
- `recentActivity`: Activity score (decays over time)
- `lastUpdate`: Timestamp for decay calculations

This enables the influence system where low player count + low activity = higher Demonlord influence.

### Bonus Application
Combat bonuses are applied in two layers:
1. Demonlord influence modifies core stats (damage dealt/taken)
2. Career tree bonuses from `getCareerSystemBonuses` are then applied on top
3. Both are multiplicative with the base damage/defense values
