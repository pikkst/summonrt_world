# T6.0.5 – Demonlord Floor Range Implementation

## Task
Implement and constrain the Demonlord floor range to 50–100 within the dungeon system.

## Findings

The game defines a maximum of 100 floors total. The Demonlord entity was originally configured with `DEMONLORD_FLOOR_MIN = 50` and `DEMONLORD_FLOOR_MAX = 100`, but the helper `getDemonlordForFloor()` returned `true` for **any** floor ≥ 50, including floors beyond 100. This was inconsistent with the actual dungeon floor count.

## Changes Made

### 1. `src/core/demonlord.ts`
Updated `getDemonlordForFloor()` to enforce the upper bound:
```typescript
export function getDemonlordForFloor(floor: number): boolean {
  return floor >= DEMONLORD_FLOOR_MIN && floor <= DEMONLORD_FLOOR_MAX;
}
```
Previously it returned `floor >= DEMONLORD_FLOOR_MIN`, which incorrectly treated floors 101+ as Demonlord floors.

### 2. `src/test/demonlord.test.ts`
Updated the `getDemonlordForFloor` test suite:
- Removed the assertion that floors above 100 return `true`.
- Added explicit assertions that floors 101 and 150 return `false`.

## Verification

Run the demonstrator (Demonlord) unit tests:
```bash
npx vitest run src/test/demonlord.test.ts
```

All tests pass, confirming that:
- Floors 50–100 are recognized as Demonlord-eligible floors.
- Floors below 50 are not Demonlord-eligible.
- Floors above 100 are not Demonlord-eligible.
