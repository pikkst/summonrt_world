# Task T4.4 - Implement Dual-Element Roll

## What was found
The `rollAffinity()` function in `src/stores/game/helpers.ts` had incorrect probabilities and no weighted element pair selection:

1. **Incorrect probabilities**: Previous implementation used `rand < 0.001` for triple element (0.1%) and `rand < 0.011` for dual element (~1%), which did not match GDD specifications.
2. **No weighted pair selection**: Secondary elements were chosen with uniform random distribution, ignoring synergistic, neutral, or opposing relationships between elements.
3. **GDD requirement**: Dual Element probability must be 1/1,000 (0.1%), Triple Element 1/1,000,000 (0.0001%), with weighted secondary element selection.

## What was done
1. **Updated `rollAffinity()` probabilities in `src/stores/game/helpers.ts`**:
   - Triple Element: `rand < 0.000001` (1 in 1,000,000)
   - Dual Element: `rand < 0.001` (1 in 1,000)
   - Single Element: default (999 in 1,000)

2. **Added `getElementPairWeight()` function**:
   - Defines synergistic pairs with weight 2: Fire+Air (Storm), Water+Ice (Glacier), Earth+Nature, Air+Lightning, Lightning+Iron
   - Defines opposing pairs with weight 0.5: Fire+Ice, Fire+Water, Light+Darkness, Nature+Fire, Ice+Fire, Water+Fire
   - All other pairs default to neutral weight 1

3. **Added `pickWeightedElement()` helper**:
   - Accepts primary element and candidate secondary elements
   - Performs weighted random selection based on `getElementPairWeight()`
   - Used for both dual-element and triple-element secondary/tertiary selection

4. **Updated dual-element logic**:
   - Secondary element is no longer uniformly random
   - Synergistic pairs are 4x more likely than opposing pairs (weight 2 vs 0.5)
   - Neutral pairs maintain baseline probability (weight 1)

## Summary
The elemental affinity system now correctly implements the GDD-specified dual-element roll with 1/1,000 probability and weighted pair selection. Players have a fair chance at synergistic element combinations while opposing pairs remain rare.
