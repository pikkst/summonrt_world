# Sprint 6 – T6.12: Test Full Clear World 10 Dungeon Simulation (Integration Test)

## Objective
Implement an integration test that simulates a full clear of a World 10 dungeon tower, validating that the entire dungeon generation and combat resolution pipeline works end-to-end from the first floor entrance to the final boss defeat.

## What Was Found

### Existing Implementation
The dungeon generation system was refactored into `src/core/dungeon/*` modules via the dungeonGenerator refactor commit. The following components were already in place:

- `generateDungeonTower(worldIndex, globalSeed)` — Builds a complete vertical tower of procedurally generated floors
- `generateDungeonFloor(worldIndex, floorIndex, globalSeed)` — Generates individual maze-based floors with entrance, boss, and treasure rooms
- `generateBossFloor(worldIndex, floorIndex, globalSeed)` — Creates an open boss arena on the final floor
- `resolveAutomatedCombat(teamA, teamB, options)` — Resolves turn-based automated combat between creature teams
- `findShortestPath(graph, startId, endId)` — Pathfinding utility for dungeon traversal
- `calculateRoomDistanceMap(graph, startId)` — BFS distance map used for room traversal ordering

### Requirements
From the task specification:
1. Simulate a complete dungeon run through World 10
2. Validate that every floor is traversable from entrance to boss
3. Verify combat encounters can be resolved on each floor
4. Confirm the final boss is defeatable
5. Ensure deterministic dungeon generation with seeded RNG

## What Was Done

### New Test File: `src/test/dungeonFullClear.test.ts`
Created a comprehensive integration test suite with 8 test cases:

#### Test Suite: `T6.12: World 10 full clear dungeon integration test`

**Test 1: `generates a complete World 10 dungeon tower`**
- Generates a World 10 tower using `generateDungeonTower(10, 12345)`
- World 10 produces 13 total floors (`3 + WorldIndex`)
- Validates all floors contain rooms, entrance, and boss room IDs

**Test 2: `is deterministic for the same seed`**
- Generates two towers with identical parameters
- Uses deep equality (`toEqual`) to verify identical output

**Test 3: `ensures every floor has a valid entrance-to-boss path`**
- Calls `findShortestPath` for each floor
- Fails with descriptive message if any floor is disconnected

**Test 4: `verifies no disconnected rooms on any floor`**
- Uses `calculateRoomDistanceMap` for BFS reachability
- Validates every room is reachable from the entrance

**Test 5: `links every floor vertically to the next floor entrance`**
- Checks `tower.verticalLinks` count equals `totalFloors - 1`
- Verifies each link connects current floor boss room to next floor entrance

**Test 6: `clears every floor with a viable party`**
- Simulates a 3-creature party (Fire, Water, Nature element coverage)
- Traverses rooms in BFS order (distance from entrance)
- Resolves combat encounters for `combat` and `elite` rooms using `resolveAutomatedCombat`
- Applies trap damage to random party members for `trap` rooms
- Grants 15% health regeneration between rooms
- Defeats each floor's boss at the end of the floor
- Verifies party survives all 13 floors

**Test 7: `produces a valid DungeonRun export for online sync`**
- Calls `exportDungeonRun(tower)` to create a `DungeonRun` object
- Validates run metadata (worldIndex, totalFloors, tower reference)
- Tests JSON serialization/deserialization for online synchronization

**Test 8: `validates safe floors every 10th floor for World 10`**
- Verifies floor 10 is marked as a safe floor
- Confirms safe floor has rest and vendor rooms
- Ensures the 10th floor is not a boss floor

### Party Composition
The test uses a balanced 3-creature party scaled for World 10 difficulty:

| Creature | Element | Level | Attack | Defense | Health |
|----------|---------|-------|--------|---------|--------|
| p1 | fire | 15 | 40 | 20 | 180 |
| p2 | water | 15 | 38 | 19 | 170 |
| p3 | nature | 15 | 36 | 18 | 160 |

### Enemy Scaling Formula
Encounters scale dynamically based on floor progression:
- `enemyLevel = max(1, 10 + floor(worldIndex / 2) + distanceFromEntrance)`
- World 10 enemies range from level 15 (entrance) to level 20+ (deep rooms and boss)
- Boss enemies receive World 10 scaling: `attack = 20 + 30 = 50`, `health = 120 + 150 = 270`
