# Task T3.6 - Implement Affinity Bonus XP

## What was found
The existing codebase had the `getWorldModifier` function implemented in src/core/xpCurve.ts for world-based XP bonuses, but lacked the affinity-based XP bonus system. Elemental advantages were already defined in combatModule.ts and missionQueue.ts for damage calculations, but XP bonuses based on elemental affinity were not implemented. The task required implementing the formula: 1.15x XP for same-element encounters and 0.85x XP for opposing-element encounters.

## What was done
- Extended src/core/xpCurve.ts with:
  - `Element` type export covering all 13 elements (fire, water, earth, air, lightning, iron, nature, ice, light, darkness, void, starlight, chaos)
  - `ELEMENTAL_OPPOSITIONS` constant mapping each element to its opposing elements
  - `getAffinityBonusXP(atkElement, defElements)` function returning:
    - 1.15 for same element match (attacker's element in defender's elements)
    - 0.85 for opposing element match (defender has an element that opposes attacker's element)
    - 1.0 for neutral element match
  - `calculateEncounterXP(baseXp, monsterLevel, worldModifier, atkElement, defElements)` function combining all XP modifiers: `(MonsterBaseXP × MonsterLevel) × WorldMod × AffinityBonus`

- Added tests to src/test/xpCurve.test.ts:
  - Tests for `getAffinityBonusXP`: same element bonus, opposing element penalty, neutral matches, edge cases (undefined/null elements)
  - Tests for `calculateEncounterXP`: combined modifiers, opposing penalty, neutral case, undefined elements fallback

- All 25 tests pass successfully using vitest
