# Task T4.13 – Failed capture → aggressive encounter or persistent territorial hostility

## What was found

The existing capture failure logic in `src/stores/game/modules/missionModule.ts` had a binary outcome:

- If the player had creatures in their party, a failed capture immediately triggered combat via `startCombat()`.
- If the player had NO creatures, the failure simply logged a warning message and the creature disappeared.

There was no persistence for hostile creatures, no distinction between a random encounter and an enraged/territorial one, and no stat differentiation for aggressive encounters.

## What was done

### 1. Added `TerritorialHostilityEntry` interface to `src/types/game.ts`

- New interface stores the full creature snapshot for hostile encounters.
- Includes `hostilityTurns` to track how long the creature remains territorial (5–10 turns).
- Stored in `PlayerState.territorialHostilities` keyed by tile coordinate.

### 2. Extended `CombatState` with `encounterType`

- Added `encounterType?: 'normal' | 'aggressive' | 'territorial'` to `CombatState`.
- Allows combat logic to know when an encounter is a normal wild spawn, an aggressive post-failure encounter, or a territorial hostility.

### 3. Updated `startCombat` in `src/stores/game/modules/combatModule.ts`

- New optional parameter `encounterType` defaults to `'normal'`.
- **Aggressive / Territorial encounters** apply stat boosts:
  - +20% max HP
  - +15% base attack
  - +10% base defense
- Custom log messages differentiate encounter types:
  - Territorial: "A territorial [name] blocks your path! It remembers your failed attempt and attacks with fury!"
  - Aggressive: "A wild [name] appears! It looks enraged and ready to fight!"

### 4. Updated `handleVictory` in `combatModule.ts`

- When a territorial encounter is won, the corresponding `territorialHostilities` entry is automatically removed from the player's state.
- Logs: "The territorial [name] has been driven off!"

### 5. Rewrote `finishCapture` failure branch in `missionModule.ts`

- **Player has creatures**: starts an aggressive encounter with boosted stats.
- **Player has no creatures**: the creature becomes territorial and is stored in `player.territorialHostilities` at the player's current tile.

### 6. Added territorial hostility check in `finishMovement` in `missionModule.ts`

- On every tile movement, all hostility entries are decremented by 1 turn; expired entries are removed.
- If the destination tile has an active territorial hostility, the player encounters the stored creature in a territorial combat before normal random encounters are rolled.
- Territorial encounter takes priority over random encounters.

### 7. Initialized `territorialHostilities` in player creation/loading

- `initGame` in `playerModule.ts` initializes `territorialHostilities: {}`.
- `loadGame` preserves `territorialHostilities` from save data.

### 8. Updated `GameActions` type signature in `stores/game/types.ts`

- `startCombat` now accepts the optional third parameter `encounterType`.

## Behavioral Changes

| Scenario | Old Behavior | New Behavior |
|---|---|---|
| Capture fails, player has creatures | Normal combat (default stats) | Aggressive combat (+20% HP, +15% ATK, +10% DEF) |
| Capture fails, player has no creatures | Log warning, creature lost forever | Creature stored as territorial hostility at current tile |
| Player moves to hostile tile | Normal movement + random encounter | Territorial encounter triggers first; hostility persists if player flees/defeated |
| Player wins territorial combat | Normal victory | Hostility removed; creature is driven off |
| Hostility expires | N/A | Removed automatically after 5–10 player turns |
