# Sprint 6 – T6.1: Fix Combat Damage Formula

## Objective
Align all combat damage calculations to the GDD-specified formula:
```
damage = (ATK − DEF × 0.5) × elementalFactor + random(−2…+2)
```

## What Was Found

### combatModule.ts (`src/stores/game/modules/combatModule.ts`)
The main interactive combat system had **four** damage formulas that deviated from the spec:

| Location | Old Formula | Issue |
|----------|-------------|-------|
| `attackWithCreature` (line 178) | `(ATK − DEF) + random(0..4)` | Missing `× 0.5` on DEF, missing elementalFactor, wrong random range |
| `useSkill` (line 227) | `(ATK × skillMult − DEF) + random(0..4)` | Missing `× 0.5` on DEF, missing elementalFactor, wrong random range |
| `triggerEnemyTurn` skill (line 287) | `(ATK × skillMult − DEF + random(0..3)) × elementalFactor` | Missing `× 0.5` on DEF, random applied before multiplier instead of after |
| `triggerEnemyTurn` basic (line 292) | `(ATK − DEF + random(0..3)) × elementalFactor` | Missing `× 0.5` on DEF, random applied before multiplier instead of after |

### combatSystem.ts (`src/modules/combat/combatSystem.ts`)
The legacy combat simulator had two formulas missing elementalFactor and the final `random(−2…+2)`:

| Location | Old Formula | Issue |
|----------|-------------|-------|
| `runCombat` attacker (line 24) | `(ATK_with_random − DEF × 0.5)` | Random baked into ATK instead of added to final damage, no elementalFactor |
| `runCombat` defender (line 33) | `(ATK_with_random − DEF × 0.5)` | Same issues as attacker |

### missionQueue.ts (`src/core/missionQueue.ts`)
The automated combat system already matched the spec:
```
damage = (ATK − DEF × 0.5) × elementalFactor + random(−2…+2)
```
No changes were needed here.

## What Was Done

### combatModule.ts changes
1. **Player basic attack** (`attackWithCreature`):
   - Changed DEF scaling from `1×` to `0.5×`
   - Moved `random(−2…+2)` to the final damage calculation
   - Kept `elementalFactor` and existing multipliers (primordial/omni/affection/career)

2. **Player skill attack** (`useSkill`):
   - Changed DEF scaling from `1×` to `0.5×`
   - Moved `random(−2…+2)` to the final damage calculation
   - Kept `skillMult`, `elementalFactor`, and existing multipliers

3. **Enemy skill attack** (`triggerEnemyTurn`):
   - Changed DEF scaling from `1×` to `0.5×`
   - Moved `random(−2…+2)` to the final damage calculation (after elementalFactor)

4. **Enemy basic attack** (`triggerEnemyTurn`):
   - Changed DEF scaling from `1×` to `0.5×`
   - Moved `random(−2…+2)` to the final damage calculation (after elementalFactor)

### combatSystem.ts changes
1. Added local `ELEMENTAL_ADVANTAGES` and `ELEMENTAL_DISADVANTAGES` tables matching `combatModule.ts`
2. Added `getElementalEffectiveness()` helper
3. Removed random from ATK power calculation
4. Applied `× elementalFactor` and added `random(−2…+2)` to final damage for both attacker and defender
