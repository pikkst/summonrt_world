# T6.14 – Integrate Career Passives into Combat

## Sprint 6 Task: T6.14

### Objective
Integrate career tree passives into all combat resolution paths, ensuring Summoner path `capture_bonus_pct`, Warrior path `damage_dealt_pct`, and Guardian path `damage_taken_pct` are consistently applied across manual combat, demonlord combat, and automated combat.

### Findings

#### Pre-existing Integrations
Manual combat (`src/stores/game/modules/combatModule.ts`) already integrated the following career passives:
- **`damage_dealt_pct`** — Applied via `applyCombatCareerBonuses()` to all player attacks and skill damage.
- **`damage_taken_pct`** — Applied via `applyDamageTakenReduction()` to all incoming enemy damage.
- **`capture_bonus_pct`** — Applied via `applyCaptureRateBonus()` in the capture system (`src/stores/game/modules/missionModule.ts`).

#### Bug Found: Incorrect `damage_taken_pct` Formula
The `applyDamageTakenReduction()` function in `src/data/careerTreeIntegration.ts` used an inverted formula:
```typescript
// Buggy version
const damageTakenMult = 1 - ((bonuses.damage_taken_pct || 0) / 100);
```
Because `damage_taken_pct` values in the career tree are **negative** for damage reduction (e.g., `-5` for 5% less damage taken), the formula produced:
- `damage_taken_pct: -5` → `1 - (-5/100) = 1.05` → **5% MORE damage taken** (incorrect)

The same bug existed in `src/stores/game/modules/demonlordModule.ts` line 118:
```typescript
// Buggy version
const withCareerDefense = 1 - ((careerBonuses.damage_taken_pct || 0) / 100);
```

#### Missing Integration: Automated Combat
`resolveAutomatedCombat()` in `src/core/missionQueue.ts` had no career passive integration. The function performed raw elemental effectiveness calculations but ignored `damage_dealt_pct` and `damage_taken_pct` entirely.

### Changes Made

#### 1. Fixed `damage_taken_pct` Formula in `careerTreeIntegration.ts`
**File:** `src/data/careerTreeIntegration.ts`
**Line:** 123

```typescript
// Corrected version
const damageTakenMult = 1 + ((bonuses.damage_taken_pct || 0) / 100);
```

This now correctly interprets negative `damage_taken_pct` values as damage reduction:
- `damage_taken_pct: -5` → `1 + (-5/100) = 0.95` → **5% LESS damage taken** (correct)
- `damage_taken_pct: 0` → `1 + 0 = 1.0` → no change

#### 2. Fixed `damage_taken_pct` Formula in `demonlordModule.ts`
**File:** `src/stores/game/modules/demonlordModule.ts`
**Line:** 118

```typescript
// Corrected version
const withCareerDefense = 1 + ((careerBonuses.damage_taken_pct || 0) / 100);
```

#### 3. Integrated Career Passives into `resolveAutomatedCombat()`
**File:** `src/core/missionQueue.ts`
**Lines:** 130-134, 178, 211

Added optional career bonus parameters:
```typescript
options?: {
  rngSeed?: number;
  worldLayer?: number;
  atkElements?: Element[];
  primordialPct?: number;
  omniPct?: number;
  damageDealtPct?: number;
  damageTakenPct?: number;
}
```

Applied bonuses to damage calculations:
- **Player team (teamA) attacks:** multiplied by `1 + (damageDealtPct / 100)`
- **Enemy team (teamB) attacks:** multiplied by `1 + (damageTakenPct / 100)`

### Career Path Mapping

| Task Designation | Career Nodes | Stat Key | Effect |
|------------------|--------------|----------|--------|
| **Summoner path** | Summoner minor/elite/legendary nodes | `capture_bonus_pct` | Increases creature capture chance |
| **Warrior path** | Blacksmith (War Engineer, Master of Alloys), Official (War Marshal), Broker (Combat Quartermaster), Summoner (Creature Warden) | `damage_dealt_pct` | Increases outgoing combat damage |
| **Guardian path** | Blacksmith (Master of Alloys), Official (War Marshal), Broker (Combat Quartermaster), Summoner (Creature Warden) | `damage_taken_pct` | Reduces incoming combat damage |

> Note: `damage_dealt_pct` and `damage_taken_pct` are not exclusive to a single career category. They are cross-career bonuses available through Blacksmith, Official, Broker, and Summoner paths.
