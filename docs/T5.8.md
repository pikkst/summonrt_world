# Task T5.8 – Add `capturePool` and `fusionRecipe` auto-generation for new creatures

## What was found

The existing creature generation system (`src/modules/creatures/creatureFactory.ts`) created `CreatureTemplate` objects without any metadata recording capture eligibility or lineage. Similarly, the fusion system in `src/stores/game/modules/missionModule.ts` created fused `CreatureInstance` objects without recording their parentage.

Specifically:

1. **`CreatureTemplate`** had no `capturePool` field, making it impossible to query which elements or conditions make a creature capturable
2. **`CreatureInstance`** had no `fusionRecipe` field, making it impossible to trace how a fused creature was created
3. When `generateCreatureTemplate()` or `registerSpeciesLine()` produced a new template, no auto-generation of capture metadata occurred
4. When `breedCreatures()` fused two creatures, only a synthetic `templateKey` was set on the child, with no explicit parent record

## What was done

### 1. Updated `src/types/game.ts`

Added optional properties to the core creature types:

**`CreatureTemplate`** (line 60):
```typescript
capturePool?: {
  compatibleElements: Element[];
};
```

**`CreatureInstance`** (line 129):
```typescript
fusionRecipe?: {
  parentAKey: string;
  parentBKey: string;
};
```

### 2. Auto-generated `capturePool` in `src/modules/creatures/creatureFactory.ts`

Both `generateSpeciesTemplate()` (line 114) and `createRandomTemplate()` (line 184) now compute and attach `capturePool` on every new `CreatureTemplate`:

```typescript
capturePool: {
  compatibleElements: [...new Set(elements)],
},
```

- `compatibleElements` is the deduplicated list of the creature's own elements
- This directly maps to the elemental affinity system: matching any `compatibleElements` with the player's own elements grants `same-element` affinity weight (1.0x) during capture
- For species line templates, the pool is derived from the line's defined element list
- For random templates, the pool is derived from the randomly rolled element list

### 3. Auto-generated `fusionRecipe` in `src/stores/game/modules/missionModule.ts`

The `breedCreatures()` function (line 749) now attaches `fusionRecipe` to every new fused `CreatureInstance`:

```typescript
fusionRecipe: {
  parentAKey: c1.templateKey,
  parentBKey: c2.templateKey,
},
```

- Records the exact parent template keys that produced the offspring
- Enables reverse-lookup queries (e.g., "show me all creatures born from this fusion pair")
- Only set on fused creatures; base creatures captured from the wild do not have `fusionRecipe`
