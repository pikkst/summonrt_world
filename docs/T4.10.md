# Task T4.10 – Implement exact capture formula

## What was found

The existing capture system in `src/stores/game/modules/missionModule.ts` used an outdated formula that did not match the GDD specification:

- The original formula used a fixed `baseChance = 0.60` that was unrelated to the GDD formula
- Rarity penalties were hardcoded in a switch statement instead of using constants
- The GDD formula `P_capture = (1 − (MonsterHP / MaxHP)) × AffinityWeight × RarityPenalty × (1 − (LevelDiff × 0.02))` was not implemented
- Affinity weight values (same/neutral/opposing) did not exist as constants
- The `ELEMENT_OPPOSITIONS` mapping was inconsistent across files

## What was done

### 1. Added capture formula constants to `src/data/constants.ts`

- **`AFFINITY_WEIGHT`** — Defined weights: `same: 1.0`, `neutral: 0.3`, `opposing: 0.1`
- **`RARITY_PENALTY`** — Defined penalties per GDD: Common 1.0, Uncommon 0.8, Rare 0.5, Epic 0.25, Legendary 0.2, Mythical 0.15
- **`ELEMENT_OPPOSITIONS`** — Added element opposition mapping for determining affinity weight
- **`getAffinityWeight(playerElements, creatureElements)`** — Function returning correct weight based on element relationship
- **`calculateCaptureFactors(...)`** — Returns all factors (hpFactor, affinityWeight, rarityPenalty, levelFactor)
- **`calculateBaseCaptureProbability(...)`** — Implements the exact GDD formula

### 2. Updated `capturing` state type in `src/stores/game/types.ts`

- Added `currentHealth` field to track creature's wounded state during capture
- Creature starts at 50-100% HP (randomized) to make the HP factor meaningful

### 3. Updated `captureCreature` in `src/stores/game/modules/missionModule.ts`

- Creature now starts at randomized wounded HP (50-100% of base health)
- This reflects the GDD intention that capture chance increases as creature HP decreases

### 4. Updated `finishCapture` in `src/stores/game/modules/missionModule.ts`

- Replaced manual probability calculation with `calculateBaseCaptureProbability()`
- Uses exact formula: `hpFactor * affinityWeight * rarityPenalty * levelFactor`
- Added career bonuses for wild_whisper (+0.15), beastmaster (+0.3), and summoner passives
- Maintained 95% hard cap on capture chance with career bonuses

### 5. Added tests in `src/test/missionQueue.test.ts`

- Tests for `AFFINITY_WEIGHT` constant values
- Tests for `RARITY_PENALTY` constant values and ordering
- Tests for `getAffinityWeight()` with same/neutral/opposing element combinations

## Formula Implementation

```typescript
// P_capture = (1 − (MonsterHP / MaxHP)) × AffinityWeight × RarityPenalty × (1 − (LevelDiff × 0.02))
export function calculateBaseCaptureProbability(
  currentHp: number,
  maxHp: number,
  playerElements: string[],
  creatureElements: string[],
  creatureClass: string,
  playerLevel: number,
  creatureWorldLevel: number
): number {
  const hpFactor = 1 - (currentHp / maxHp);
  const affinityWeight = getAffinityWeight(playerElements, creatureElements);
  const rarityPenalty = RARITY_PENALTY[creatureClass] ?? 1.0;
  const levelDiff = Math.max(0, creatureWorldLevel - playerLevel);
  const levelFactor = Math.max(0.1, 1 - (levelDiff * 0.02));
  
  return hpFactor * affinityWeight * rarityPenalty * levelFactor;
}
```
