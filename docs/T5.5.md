# Task T5.5 - Implement Procedural Identity

## What was found
The fusion system (T5.1-T5.4) was already implemented with:
- Fusion matrix with 10x10 element combinations (T5.1) in `src/data/fusionMatrix.ts`
- Skill inheritance system (T5.3) in `src/data/fusionUtils.ts`
- Trait synergies (T5.4) in `src/data/traitSynergy.ts`

However, the procedural identity system for fused creatures was missing. When creatures are fused via the `breedCreatures` function in `src/stores/game/modules/missionModule.ts`, there was no visual identity generation for the resulting creature.

## What was done
1. **Created `src/data/proceduralIdentity.ts`**:
   - Defined `ProceduralIdentity` interface with `headVariant`, `bodyVariant`, `limbVariant`, `elementalFx`, and `colorPalette` fields
   - Defined variant arrays for each creature type (beast, dragon, undead, construct, spirit, demon, celestial, insect, plant):
     - `HEAD_VARIANTS`: 8 head variations per type
     - `BODY_VARIANTS`: 8 body variations per type
     - `LIMB_VARIANTS`: 8 limb variations per type
   - Defined elemental FX mapping (`ELEMENTAL_FX`) for all elements including quest-only elements (void, starlight, chaos, omni)
   - Defined base colors (`BASE_COLORS`) for all elements
   - Implemented `interpolateColor` function for color palette interpolation
   - Implemented `generateColorPalette` function that creates primary/secondary/accent colors based on creature elements
   - Implemented `generateProceduralIdentity` main function that generates randomized visual identity based on creature type and elements
   - Implemented `getCombinedElementalFX` for dual-element creatures to combine FX from both elements

2. **Updated `src/types/game.ts`**:
   - Added `ProceduralIdentity` interface definition
   - Added `proceduralIdentity?: ProceduralIdentity` field to `CreatureInstance` interface

3. **Updated `src/stores/game/modules/missionModule.ts`**:
   - Imported `generateProceduralIdentity` from the new module
   - Integrated procedural identity generation into `breedCreatures` function (line ~734) - assigns a unique visual identity to each fused creature
   - Integrated procedural identity generation into `finishCapture` function (line ~444) - assigns visual identity to newly captured creatures

4. **Created `src/test/proceduralIdentity.test.ts`**:
   - Tests for `interpolateColor` function
   - Tests for `generateColorPalette` with single and dual elements
   - Tests for `generateProceduralIdentity` for all creature types
   - Tests for `getCombinedElementalFX` with single and dual elements
   - Tests verifying variant arrays exist for all creature types
   - Tests verifying elemental FX and base colors exist for all elements

## Summary
The procedural identity system now provides:
- **Head/Body/Limb Variations**: Each creature type has 8 distinct visual variants for each body part category, selected randomly via RNG
- **Elemental FX**: Trail, aura, and impact effects mapped to each element
- **Color Palette Interpolation**: Colors for primary and secondary elements are interpolated with an accent color based on a random factor
- **Integration**: All fused and captured creatures now receive a unique `proceduralIdentity` property that can be used by UI components for visual rendering
