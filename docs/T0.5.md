## Changes Made

### 1. Updated Type Imports
Added `CreatureTemplate` and `Skill` to the import statement:
```typescript
import type { PlayerState, WorldData, LogEntry, ElementalAffinity, Screen, CombatState, DungeonState, CreatureInstance, CreatureTemplate, Skill, QuestInstance, Element, InventoryStack, CommunityState, CommunityPlayer } from '../types/game.ts';
```

### 2. Fixed `CombatState.enemyTemplate` Type
Changed from `any` to `CreatureTemplate | null`:
```typescript
export interface CombatState {
  // ...
  enemyTemplate?: CreatureTemplate | null;
}
```

### 3. Created `ServerPlayer` Interface
Replaced `any` in `toCommunityPlayer` with a proper interface:
```typescript
interface ServerPlayer {
  _id?: string;
  id?: string;
  username?: string;
  name?: string;
  level?: number;
  archetype?: string;
  currentWorld?: number;
  currentWorldId?: number;
  posX?: number;
  posY?: number;
  x?: number;
  y?: number;
  lastActive?: string;
  lastSeen?: string;
  isOnline?: boolean;
  online?: boolean;
  skillsUnlocked?: Record<string, boolean>;
  skillPoints?: number;
  settings?: { musicVolume?: number; sfxVolume?: number; showLogTimestamps?: boolean };

```

### 4. Created `Landmark` Interface
Replaced `any` for landmark data:
```typescript
interface Landmark {
  x: number;
  y: number;
  type: string;
}
```

### 5. Added Null Checks
Added optional chaining (`?.`) to `enemyTemplate` accesses in `triggerEnemyTurn` and `handleVictory` functions to handle the `null` case safely.

### 6. Fixed Skill Mapping
Changed skill mapping in boss egg hatching to properly handle both `Skill` objects and string keys:
```typescript
skills: template.skills.map((s: Skill | string) => typeof s === 'string' ? s : s.key),
```

### 7. Fixed `addPlayerXP` Type
Changed `type: any` to `type: LogEntry['type']`:
```typescript
export function addPlayerXP(player: PlayerState, xpGained: number, appendLog: (text: string, type: LogEntry['type']) => void): PlayerState
```

### 8. Fixed `GameShell.tsx` Attack Command
Created a proper `CreatureTemplate` for the training dummy attack command instead of using a partial object.
