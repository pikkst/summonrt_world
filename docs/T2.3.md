# Task T2.3 – Implement ActiveMissionQueue Zustand Slice

## What was found

The `GameStoreState` in `src/stores/game/types.ts` had no `missions` field. The existing `missionModule.ts` provided immediate-action helpers (`movePlayer`, `finishMovement`, `searchArea`, `captureCreature`, etc.) but contained no state for a time-based mission queue. The `src/core/missionQueue.ts` file created in T2.1 already exported `ActiveMission`, `MissionType`, `MissionStatus`, `createActiveMission`, `isMissionExpired`, and `getRemainingSeconds`, but none of these were wired into the Zustand store.

The store was missing the core queue operations required by Sprint 2:
- `missions[]` array to track active missions
- `addMission()` to enqueue a new timed mission
- `completeMission()` to mark a mission as successfully finished
- `failMission()` to mark a mission as failed

## What was done

### 1. Extended `GameStoreState` with mission queue
Added `missions: ActiveMission[]` to the store state interface in `src/stores/game/types.ts:86`.

### 2. Added new actions to `GameActions`
Extended the store action interface with three methods:
- `addMission(mission: ActiveMission)`: appends a new mission to the queue
- `completeMission(missionId: string)`: sets the matching mission status to `COMPLETED`
- `failMission(missionId: string)`: sets the matching mission status to `FAILED`

### 3. Implemented the actions in `missionModule.ts`
Created the three reducer functions inside the existing `missionActions` factory:
- `addMission` uses `set(state => ({ missions: [...state.missions, mission] }))` to append
- `completeMission` maps over `missions` and replaces the target item with an updated copy where `status` is `'COMPLETED'`
- `failMission` maps over `missions` and replaces the target item with an updated copy where `status` is `'FAILED'`

Both mutating functions preserve all other mission fields immutably by spreading the original object.

### 4. Initialized store state
Updated `initialGameState` in `src/stores/gameStore.ts:46` to include `missions: []`, ensuring the queue starts empty on game initialization.

### 5. Integrated types
Imported `ActiveMission` from `../../core/missionQueue.ts` into `src/stores/game/types.ts` and re-exported it so it is accessible wherever `GameStore` types are consumed.
