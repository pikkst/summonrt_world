# Task T4.7 - Fix generateCreatureTemplate() to use GDD Rarity Weights

## What was found

The `generateCreatureTemplate()` function in `src/modules/creatures/creatureFactory.ts` was using incorrect rarity weights for creature generation:

1. **Incorrect weights in `creatureFactory.ts`**: The `weightedRandomClass()` function had weights `[55, 28, 12, 4, 0.9, 0.1]` instead of the GDD-specified `[60, 25, 10, 4, 0.9, 0.1]`
2. **Inconsistent CLASS_WEIGHTS constant**: The `src/data/constants.ts` file had `CLASS_WEIGHTS = [55, 28, 12, 4, 0.9, 0.1]` which did not match the GDD specification
3. **No test coverage**: There were no tests validating the creature rarity distribution matches the GDD specification

## What was done

### 1. Updated CLASS_WEIGHTS in `src/data/constants.ts`
- Changed `CLASS_WEIGHTS` from `[55, 28, 12, 4, 0.9, 0.1]` to `[60, 25, 10, 4, 0.9, 0.1]`
- This now matches the GDD specification: Common 60%, Uncommon 25%, Rare 10%, Epic 4%, Legendary 0.9%, Mythical 0.1%

### 2. Refactored `weightedRandomClass()` in `src/modules/creatures/creatureFactory.ts`
- Removed hardcoded weights array and now references the imported `CLASS_WEIGHTS` constant
- Added import for `CLASS_WEIGHTS` from `../../data/constants.ts`
- This ensures consistency if weights are ever updated in one place

### 3. Created test file `src/test/creatureFactory.test.ts`
- Added distribution test verifying creature classes match GDD weights within tolerance
- Added mythical rarity test (100,000 samples) confirming ~0.1% occurrence
- Added boss creature test confirming bosses are epic/legendary/mythical tier
- Added stat validation test ensuring all base stats are positive
- Added element validation test confirming elements are from the starter pool

## Summary

The creature generation system now uses the correct GDD rarity weights:
- **Common**: 60% (increased from 55%)
- **Uncommon**: 25% (decreased from 28%) 
- **Rare**: 10% (increased from 12%)
- **Epic**: 4% (unchanged)
- **Legendary**: 0.9% (unchanged)
- **Mythical**: 0.1% (unchanged)

All tests pass and the TypeScript type check completes without errors.
