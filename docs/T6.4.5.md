# Sprint 6 - T6.4.5: Create generateDungeonTower(worldIndex)

## Objective
Create `generateDungeonTower(worldIndex)` for the central Sword Art Online-style dungeon tower.

Required behavior:
- Build one continuous vertical tower.
- Floor count equals `BaseFloors + WorldIndex`.
- Link each floor vertically from its exit room to the next floor entrance.
- Mark every 10th floor as a safe floor with rest area, vendor, and teleport unlock metadata.

## What Was Found

### Existing Dungeon Foundation
The dungeon generator already had the required floor-level systems in `summoner-world/src/core/dungeonGenerator.ts`:

- `generateDungeonFloor(worldIndex, floorIndex, globalSeed)` creates deterministic connected floor graphs.
- Floor graphs already include entrance and boss/exit room IDs.
- Room assignment already supports rest and vendor rooms.
- Treasure placement, multi-path validation, deterministic floor seeds, and pathfinding utilities were already implemented.

### Relevant Design Documentation
`docs/Sumoner_World_dec/15_Dungeon_Bible.md` defines the central tower rules:

- Every world contains a central dungeon tower.
- The tower progresses vertically.
- The exit of one floor links to the entrance of the next floor.
- Every 10th floor is safe.
- Safe floors provide rest, vendor, and teleport unlock support.

## What Was Done

### Added Tower Types
Updated `summoner-world/src/types/game.ts` with:

- `DungeonTowerVerticalLink`
- `DungeonTowerSafeFloor`
- `DungeonTower`

These types keep floor room IDs local to each floor while storing cross-floor progression as explicit tower metadata.

### Added Tower Generation
Updated `summoner-world/src/core/dungeonGenerator.ts` with:

- `DUNGEON_BASE_FLOORS`
- `getDungeonTowerFloorCount(worldIndex)`
- `generateDungeonTower(worldIndex, globalSeed = 12345)`

The tower generator:

1. Calculates total floors as `DUNGEON_BASE_FLOORS + worldIndex`.
2. Generates each floor using the existing deterministic floor generator.
3. Creates vertical links from each floor `bossRoomId` to the next floor `entranceRoomId`.
4. Marks every 10th floor as safe.
5. Forces safe floors to contain a real rest room and vendor room.
6. Records teleport unlock room metadata for safe floors.

### Added Safe Floor Metadata
Safe floor metadata records:

- `floorIndex`
- `restRoomId`
- `vendorRoomId`
- `teleportUnlockRoomId`

This keeps teleport unlock support available for future progression, UI, and persistence tasks without changing the existing room type union.

### Added Tests
Updated `summoner-world/src/test/dungeonGenerator.test.ts` with tower tests for:

- Floor count formula.
- Vertical exit-to-entrance links.
- Safe floor marking every 10th floor.
- Rest/vendor room enforcement on safe floors.
- Deterministic tower generation for the same world and seed.
