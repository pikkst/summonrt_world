# T0.7 - Enable tsconfig.json Strict Mode

## Task Description
Enable `tsconfig.json` strict mode (`strict: true`, `noUncheckedIndexedAccess: true`).

## Analysis

### Initial State
- `strict` was set to `false`
- `noUncheckedIndexedAccess` was not present in the configuration

### Changes Made
Updated `summoner-world/tsconfig.json`:
- Changed `strict` from `false` to `true`
- Added `noUncheckedIndexedAccess: true`

### TypeScript Errors Discovered
Enabling strict mode and `noUncheckedIndexedAccess` surfaced **42 new TypeScript errors** across the codebase:

| File | Error Count | Primary Issues |
|------|-------------|----------------|
| `src/stores/gameStore.ts` | 27 | Object possibly undefined, type mismatches |
| `src/utils/SeededRandom.ts` | 6 | Generic type `T | undefined` not assignable to `T` |
| `src/ui/GameShell.tsx` | 4 | Object possibly undefined, type mismatches |
| `src/modules/creatures/creatureFactory.ts` | 1 | Object possibly undefined |
| `src/modules/combat/combatSystem.ts` | 1 | Object possibly undefined |
| `src/ui/FusionPanel.tsx` | 1 | `string | undefined` not assignable to `string` |

#### Sample Error Patterns
1. **Object possibly undefined** (TS18048/TS2532) — accessing properties on values that may be `undefined`
2. **Type assignment mismatches** (TS2322/TS2345) — `string | undefined` assigned to `string`, `number | undefined` assigned to `number`
3. **Generic type issues** (TS2322) — `T | undefined` not assignable to `T` in utility functions

### Next Steps Required
The following tasks should address these errors:
- **T0.8** — Remove duplicate `LogEntry` type (may resolve some type mismatches)
- **T1.x+** — Further refactoring and type safety improvements across modules
