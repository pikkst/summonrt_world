# Creature AI Bible
## Part 02 — Needs System

### 1. Overview
The Needs System is the foundation of creature decision making. Every AI decision begins by evaluating internal needs before considering combat or exploration.

## Core Needs
- Hunger
- Thirst
- Fatigue
- Safety
- Territory
- Social
- Reproduction
- Curiosity

## Need Priority
Emergency needs always override normal behaviour.

Priority:
1. Survival
2. Escape danger
3. Heal
4. Hunger
5. Thirst
6. Sleep
7. Territory
8. Social
9. Exploration

## Need Values
Each need ranges from 0 to 100.

0 = Fully Satisfied
100 = Critical

## Utility Calculation

Utility = NeedWeight × CurrentNeed × PersonalityModifier × WorldModifier

The action with the highest utility becomes the current goal.

## Hunger

Sources:
- Time
- Combat
- Movement
- Cold weather

Food preferences:
- Herbivore
- Carnivore
- Omnivore
- Elemental feeder
- Magical feeder

## Thirst

Water sources:
- Rivers
- Lakes
- Rain
- Magical springs

Desert creatures consume water more efficiently.

## Fatigue

Generated by:
- Walking
- Hunting
- Fighting
- Migration

Sleeping restores fatigue and slightly heals health.

## Safety

Threat evaluation considers:
- Enemy level
- Group size
- Health percentage
- Nearby allies
- Escape routes

## Personality Influence

Brave:
- ignores fear longer

Cowardly:
- flees earlier

Curious:
- explores despite moderate hunger

Territorial:
- prioritizes defending territory.

## Configuration Example

```json
{
  "species":"Fire Wolf",
  "needs":{
    "hungerRate":0.8,
    "thirstRate":0.5,
    "fatigueRate":0.6,
    "fearThreshold":70
  }
}
```

## Acceptance Criteria
- Needs are deterministic.
- Needs persist in saves.
- Needs work offline and online.
- All thresholds configurable.
- No species-specific hardcoding.
