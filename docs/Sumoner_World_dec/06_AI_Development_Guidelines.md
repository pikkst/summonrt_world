# AI Development Guidelines

## Purpose
This document defines mandatory rules for AI-assisted development.

## Architecture Rules
- Never duplicate systems.
- Prefer composition over inheritance.
- Every system must be modular.
- Every module must be independently testable.

## Coding Rules
- TypeScript strict mode.
- English comments only.
- Functions under 100 lines where possible.
- No circular dependencies.

## Definition of Done
- Feature implemented.
- Unit tests passing.
- Integration tests passing.
- Documentation updated.
- Save/load compatible.
- MMO compatible.

## AI Workflow
1. Read GDD
2. Read Technical Specification
3. Read Master Roadmap
4. Implement only current task
5. Run tests
6. Commit
