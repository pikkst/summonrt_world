# Task T2.4 – Create heartbeat.ts with 1-second Interval Loop

## What was found

The `src/core/` directory had `missionQueue.ts` and `gameEngine.ts` but no heartbeat module for processing expired missions. The `missionModule.ts` in `src/stores/game/modules/` had `isMissionExpired` helper function but no periodic processor. The store had basic mission CRUD operations but no automated resolution mechanism. The `MissionResult` type was missing from `src/types/game.ts`.

## What was done

### 1. Added `MissionResult` type to `src/types/game.ts`
Created the schema for mission resolution outcomes with fields: `victory: boolean`, `battle_log: string[]`, `rewards: InventoryStack[]`, `xp: number`. This type will be used by T2.8's automated combat engine and T2.9's mission result schema.

### 2. Created `src/core/heartbeat.ts`
Implemented the time-based execution engine with:

- **`HeartbeatCallbacks` interface** – Abstracts store interactions for testability and future server integration:
  - `getCurrentTime()`: Returns current timestamp (defaults to `Date.now()`)
  - `getMissions()`: Returns all active missions from state
  - `removeMission(missionId)`: Removes completed mission from queue
  - `getMissionById(missionId)`: Fetches specific mission
  - `resolveMissionCallbacks`: Per-mission-type handlers (EXPLORE_TIER_1, SCOUT_DUNGEON, SMELT_ORE, CRAFT_ITEM, STORE_VISIT, TAX_EDICT, CARAVAN_ROUTE)

- **`createHeartbeat(callbacks)` factory** – Creates a managed heartbeat instance:
  - `start()`: Begins 1-second `setInterval` loop, sets `running` flag
  - `stop()`: Clears interval and stops the loop
  - `tick()`: Immediate check for expired missions (used for manual triggering and offline catch-up)

- **`resolveMission(mission, callbacks)`** – Processes expired missions:
  - Returns `MissionResult` with default values (victory: true, empty rewards/log)
  - Invokes the appropriate per-type callback for custom resolution logic
  - Calls `removeMission` to clean up completed mission from queue

- **`updateMissionStatus(mission, newStatus)`** – Returns a new mission object with updated status (immutable update pattern)

- **`processMissionResult(mission, result)`** – Extracts rewards and completion status from mission result

### 3. Created tests in `src/test/heartbeat.test.ts`
Comprehensive test coverage for:
- Heartbeat instance creation with start/stop/tick methods
- 1-second interval loop activation verification
- Expired mission processing (currentTime >= end_time + IN_PROGRESS status)
- Non-expired mission protection (no processing before end_time)
- Non-IN_PROGRESS mission protection (COMPLETED/FAILED/PENDING status ignored)
- MissionResult structure validation
- Per-mission-type callback routing
- Status update immutability
- Result processing helper
