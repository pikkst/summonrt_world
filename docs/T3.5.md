# Task T3.5 - World Modifier XP

## What was found
No world scaling existed on XP grants. The `xpCurve.ts` module (added in T3.4) had XP threshold and curve
functions but no multiplier for the active world. XP gains from combat, quests, and activities were
flat regardless of which world the player was currently in.

## What was done
- Added `getWorldModifier(worldIndex: number): number` to `src/core/xpCurve.ts` using the formula
  `WorldMod = 1 + (WorldIndex × 0.05)` (e.g. World 1 = 1.05x, World 10 = 1.50x, World 100 = 6.00x)
- Added an optional `worldModifier` parameter to `addPlayerXP()` in `src/stores/game/helpers.ts` (defaults
  to `1` for backward compatibility) and multiplies the incoming XP by the modifier before processing
  level-ups.
- Applied the world modifier to all three XP-granting paths:
  - `combatModule.ts` — handler victory XP scaled by `getWorldModifier(player.currentWorldId)`
  - `missionModule.ts` — quest reward XP scaled by `getWorldModifier(currentWorldId)`
  - `playerModule.ts` — activity/training XP scaled by `getWorldModifier(currentWorldId)`
- Added 5 new unit tests in `src/test/xpCurve.test.ts` covering the world modifier formula for Worlds 1, 10,
  100, the generic formula loop, and the invalid-index guard.
