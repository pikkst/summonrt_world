# T3.11: Turn-Count Based World-Tick System

## Overview
Implements a turn-count based world-tick system where 1 turn equals 6 minutes of game-time (240 turns per day). Triggered in the mission queue heartbeat to simulate ecosystem progression.

## Constants
- `MINUTES_PER_TURN = 6` - Each turn advances game time by 6 minutes
- `TURNS_PER_DAY = 240` - Number of turns in a full game day
- `ECOSYSTEM_BASELINE_THRESHOLD = 0.1` - Threshold for triggering overhunting
- `BIOME_SHIFT_THRESHOLD = 0.1` - Threshold for biome shifts

## Core Functions

### `calculateWorldTicks(params: WorldTickParams): WorldTickStats`
Calculates world tick progression based on elapsed real time:
- Computes elapsed minutes from `currentRealTime - lastWorldTickTime`
- Calculates turns elapsed: `Math.floor(elapsedMinutes / MINUTES_PER_TURN)`
- Updates game time minutes with wrap-around at 1440 (24 hours)
- Updates day count when full days pass

### `processWorldTick(callbacks: WorldTickCallbacks, params: WorldTickParams): WorldTickStats`
Orchestrates world tick processing:
- Calls `calculateWorldTicks` for time calculations
- Triggers `onWorldTick` callback when turns advance
- Triggers `onMissionsProgress` callback with elapsed minutes

### `updateEcosystem(params): EcosystemUpdate`
Manages ecosystem state changes:
- Tracks overhunting status (180 turns duration)
- Triggers sanctuary when resource levels drop below threshold
- Handles resource respawn (5% chance per tick for low resources, 30-90 turn cooldown)

### `getTimeUntilNextTurn(currentRealTime, lastWorldTickTime): number`
Returns milliseconds until the next turn:
- `msPerTurn = MINUTES_PER_TURN * 60 * 1000 = 360000ms`
- Uses modulo arithmetic for accurate timing

## State Integration
The world tick system integrates with `GameStore` through:
- `lastWorldTickTime: number` - Real timestamp of last tick
- `turnCount: number` - Total turns elapsed
- `player.gameTimeMinutes: number` - Current game time (0-1439)
- `player.dayCount: number` - Current game day

## Heartbeat Integration
The heartbeat (`createHeartbeat`) calls `processWorldTick` on each tick cycle:
- Checks for time-based turn progression
- Updates ecosystem state
- Triggers callbacks for UI/state updates
