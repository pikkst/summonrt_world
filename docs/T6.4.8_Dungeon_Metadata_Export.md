# T6.4.8 – Dungeon Metadata Export

## Task Summary

Add dungeon metadata export functionality to store floor graph, room types, treasure locations, and boss room IDs into a serializable `DungeonRun` object for persistence, online synchronization, and party dungeon runs.

## What Was Found

### Existing Architecture

- **Dungeon Generation**: `src/core/dungeonGenerator.ts` contains `generateDungeonFloor()` and `generateDungeonTower()` which produce `DungeonFloorGraph` and `DungeonTower` objects.
- **Dungeon Floor Graph** already contains:
  - `rooms[]` with `id`, `x`, `y`, `type`, `connections`
  - `entranceRoomId` and `bossRoomId`
  - `treasureRoomIds[]`
  - `layoutType`, `isBossFloor`, `worldElement`, `environmentalHazards`, `bossScaling`
- **Dungeon Tower** contains all floors, vertical links between floors, and safe floor metadata.
- **DungeonState** in the game store tracked active run status but did **not** hold the generated tower metadata.
- `enterDungeon()` only switched the screen to `'dungeon'` without generating or storing any dungeon layout data.

### Gap

There was no mechanism to export the generated dungeon layout into a persistence-ready format. The `DungeonState` lacked a `tower` field, and no `DungeonRun` type existed to bundle run state with the generated metadata.

## What Was Done

### 1. New Types (`src/types/game.ts`)

Added `DungeonRun` interface:

```typescript
export interface DungeonRun {
  runId: string;
  worldIndex: number;
  globalSeed: number;
  totalFloors: number;
  currentFloor: number;
  clearedFloors: number[];
  bossDefeated: boolean;
  active: boolean;
  tower: DungeonTower;
}
```

Extended `DungeonState` with optional `tower` field:

```typescript
export interface DungeonState {
  active: boolean;
  worldId: number;
  currentFloor: number;
  totalFloors: number;
  clearedFloors: number[];
  bossDefeated: boolean;
  inEncounter: boolean;
  encounterType?: 'guardian' | 'trap' | 'treasure' | 'boss';
  encounterName?: string;
  tower?: DungeonTower;
}
```

### 2. Export Function (`src/core/dungeonGenerator.ts`)

Added `exportDungeonRun()` function:

```typescript
export function exportDungeonRun(tower: DungeonTower, runId?: string): DungeonRun
```

This function:
- Accepts a `DungeonTower` and optional custom `runId`
- Returns a `DungeonRun` object bundling:
  - Run identification (`runId`, `worldIndex`, `globalSeed`)
  - Run progress state (`currentFloor`, `clearedFloors`, `bossDefeated`, `active`)
  - Full tower metadata (`tower`) including:
    - All floor graphs with room connections and types
    - Treasure room IDs per floor
    - Boss room IDs per floor
    - Entrance room IDs per floor
    - Vertical links between floors
    - Safe floor metadata (rest/vendor/teleport rooms)
    - Environmental hazards and boss scaling for boss floors

### 3. Store Integration (`src/stores/game/modules/missionModule.ts`)

Updated `enterDungeon()` to:
- Generate a deterministic tower using `generateDungeonTower(currentWorldId, globalSeed)`
- Export the run via `exportDungeonRun(tower)`
- Store the `tower` directly in `DungeonState` for immediate access during the run

The global seed is derived from `currentWorldId * 1000 + player.level`, making it deterministic per player-world context while remaining reproducible for save/load and online sync.

### 4. Store Types (`src/stores/game/types.ts`)

Added `DungeonRun` to the re-exported types so the store layer can reference it.

### 5. Tests (`src/test/dungeonGenerator.test.ts`)

Added 4 new test cases for `exportDungeonRun`:

- **Default runId generation**: Verifies a runId is auto-generated when none is provided
- **Custom runId**: Verifies provided runId is preserved
- **Full tower metadata preservation**: Validates floor graphs, room types, treasure locations, boss room IDs, and entrance room IDs across all floors
- **Serialization for online sync**: Confirms the `DungeonRun` can be JSON-stringified and parsed without data loss
- **Deterministic output**: Confirms identical towers produce identical runs

All 41 tests pass (including the 4 new ones).
