# Task T4.5 - Implement Triple-Element Roll with Primordial Trait

## What was found

The triple-element roll (1/1,000,000 probability) was already implemented in `src/stores/game/helpers.ts` as part of T4.4 work. However, the accompanying "Primordial" trait — which grants +20% all elemental damage — was missing:

1. **Triple-element roll existed**: `rollAffinity()` already checked `rand < 0.000001` and assigned `primary`, `secondary`, and `tertiary` elements with weighted pair selection.
2. **No Primordial trait**: The `ElementalAffinity` interface had no `traits` field, so there was no way to store or reference the Primordial trait on the player.
3. **No damage bonus**: Neither manual combat (`combatModule.ts`) nor automated combat (`missionQueue.ts`) applied any bonus for triple-element affinities.
4. **GDD requirement**: T4.5 specifies that a 1/1,000,000 triple-element roll should confer the "Primordial" trait, granting +20% all elemental damage.

## What was done

1. **Added `traits?: string[]` to `ElementalAffinity` interface** (`src/types/game.ts:19-25`):
   - Allows affinity records to carry arbitrary trait strings like `"primordial"`.

2. **Updated `rollAffinity()` in `src/stores/game/helpers.ts:11-16`**:
   - When `rand < 0.000001` (triple-element roll), the returned affinity now includes `traits: ['primordial']`.
   - Dual-element and single-element rolls remain trait-less.

3. **Added Primordial damage multiplier in manual combat** (`src/stores/game/modules/combatModule.ts`):
   - Introduced `getPrimordialDamageMultiplier(player)` helper that returns `1.2` when the player's affinity includes the `"primordial"` trait, otherwise `1`.
   - Applied to `attackWithCreature` (line ~138): elemental physical attacks deal +20% damage when the attacking creature's primary element exists.
   - Applied to `useSkill` (line ~181): elemental skill attacks deal +20% damage when the skill has an `element` property.
   - Non-elemental attacks (no element) are unaffected by Primordial.

4. **Added Primordial bonus support in automated combat** (`src/core/missionQueue.ts:129-242`):
   - Extended `resolveAutomatedCombat` options with `primordialPct?: number` parameter.
   - When `primoralPct` is provided (e.g., `20`), all elemental attacks by team A receive `1 + (primordialPct / 100)` multiplier.
   - Enemy team (team B) does not receive Primordial bonuses.

## Summary

The triple-element roll now fully implements the T4.5 GDD specification: a 1-in-1,000,000 chance at player creation awards the "Primordial" trait, which grants a +20% bonus to all elemental damage in both manual and automated combat. The feature is type-safe, respects existing `noUncheckedIndexedAccess` strict mode, and compiles without errors.
