# T3.10 ‚Äì Real-Time Resource Regeneration (Offline Mode)

## Summary
Implemented time-based offline regeneration for the four core player resources: `energy`, `nerve`, `happy`, and `life`. The feature calculates elapsed wall-clock time since the last resource update and restores resources proportionally when the player loads or logs in after being away.

## What Was Found
- Player resources (`energy`, `nerve`, `happy`, `life`) were defined in `src/types/game.ts` and initialized in `initGame()` and `register()`/`login()`, but there was no regeneration logic.
- Each `Resource` object already included a `lastUpdate` timestamp, but it was never read for regeneration purposes.
- UI labels in `src/ui/ResourcePanel.tsx` did not match the underlying stat keys (`Health` ‚Üí `life`, `Mana` ‚Üí `energy`, `Endurance` ‚Üí `happy`, `Willpower` ‚Üí `nerve`).
- `loadGame()` and `login()` were the two primary offline entry points where regeneration needed to be applied.

## What Was Done

### 1. `src/stores/game/helpers.ts` ‚Äî `applyResourceRegeneration`
Added a new pure function that takes a `PlayerState` and a current timestamp, reads each resource‚Äôs `lastUpdate`, and restores resources based on elapsed minutes:

| Resource | Rate per minute |
|----------|-----------------|
| energy   | 1               |
| nerve    | ~0.33 (1 per 3 min) |
| happy    | ~0.5 (1 per 2 min) |
| life     | ~0.2 (1 per 5 min) |

- Resources are capped at their `max` values.
- If `lastUpdate` is missing or invalid, that resource is skipped.
- If elapsed time is under 1 minute, no regeneration is applied.
- The function returns a new `PlayerState` with updated `current` values and a fresh `lastUpdate` timestamp on every changed resource.

### 2. `src/stores/game/modules/playerModule.ts` ‚Äî Integration Points
Wired `applyResourceRegeneration` into both offline entry points:

- **`loadGame()`**: After reconstructing the player from `localStorage`, regenerates resources before applying the state.
- **`login()`**: After constructing the player from the server response, regenerates resources before applying the state.

This ensures offline catch-up (missions, heartbeat) runs against a fully regenerated player state.

### 3. `src/ui/ResourcePanel.tsx` ‚Äî UI Label Alignment
Updated bar labels to match the actual `PlayerState` property names:

| Old Label | New Label | Property |
|-----------|-----------|----------|
| Health    | Life      | `life`   |
| Mana      | Energy    | `energy` |
| Endurance | Happy     | `happy`  |
| Willpower | Nerve     | `nerve`  |

### 4. `src/test/regeneration.test.ts` ‚Äî Coverage
Added 7 unit tests covering:
- No regeneration when `lastUpdate` is absent
- No regeneration when elapsed time < 1 minute
- Partial regeneration for each resource at 5 minutes elapsed
- Capping resources at max values
- Updating `lastUpdate` to current time
- Mixed valid/invalid `lastUpdate` values
- Future `lastUpdate` timestamps (no negative regeneration)

All 116 project tests pass.
