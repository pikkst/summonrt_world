# T5.11 – Career Passive Integration into Fusion & Full Game Loop

## Summary
Integrated all career tree passives into the core game loop: fusion, combat, capture, missions, and world tick. Expanded the career tree from 19 nodes to 73 nodes with cross-career synergies. Created a centralized integration layer (`careerTreeIntegration.ts`) that maps every node stat to specific game systems and executes aggregate bonuses on every game tick.

## What Was Done

### 1. Career Tree Expansion (`src/data/careerTree/nodes.json`)
- Expanded from **19 nodes → 73 nodes** across 6 careers
- Added **50+ new nodes** with cross-career synergy connections
- Each career now has ~10–12 nodes including minors, notables, keystones, elite crossover nodes, and legendary nodes
- Cross-career synergies implemented via connection paths:
  - **Blacksmith ↔ Broker**: Guild Artificer (caravan speed, tariff discount)
  - **Blacksmith ↔ Summoner**: War Engineer (damage dealt, fusion timer reduction)
  - **Blacksmith ↔ Explorer**: Supply Chain Master (yield bonus, exploration speed)
  - **Explorer ↔ Broker**: Trade Route Scout (caravan speed, arbitrage chance)
  - **Explorer ↔ Summoner**: Beast Guide (capture rate, affection gain)
  - **Explorer ↔ Blacksmith**: Miner's Path (yield bonus, smelting speed)
  - **Official ↔ Shopkeeper**: Market Regulator (selling price, tax revenue)
  - **Official ↔ Broker**: Trade Diplomat (tariff discount, caravan speed)
  - **Official ↔ Summoner**: War Marshal (damage dealt/taken)
  - **Shopkeeper ↔ Broker**: Market Analyst (store traffic, arbitrage)
  - **Shopkeeper ↔ Official**: Tax Auditor (tax revenue, public order)
  - **Shopkeeper ↔ Summoner**: Creature Merchant (rare sourcing, store traffic)
  - **Summoner ↔ Blacksmith**: Soul Forger (fusion success, crafting speed)
  - **Summoner ↔ Explorer**: Wilderness Tamer (capture rate, encounter avoidance)
  - **Summoner ↔ Official**: Creature Warden (public order, capture rate)

### 2. Career Tree Integration Layer (`src/data/careerTreeIntegration.ts`)
- New file defining the complete node→system mapping
- `getCareerSystemBonuses(aggregatedStats)` computes all derived bonuses from unlocked node stats
- `getCareerModifiers(aggregatedStats, playerState?)` provides backward-compatible mission modifiers
- Helper functions for applying career bonuses to specific systems:
  - `applyCombatCareerBonuses()` – damage dealt multiplier
  - `applyDamageTakenReduction()` – incoming damage reduction
  - `applyCaptureRateBonus()` – capture probability boost
  - `applyXPBoost()` – mission and creature XP boost

### 3. Integration Points

#### Combat (`src/stores/game/modules/combatModule.ts`)
- `attackWithCreature()` now applies `damage_dealt_pct` from career passives
- `useSkill()` now applies `damage_dealt_pct` from career passives
- `triggerEnemyTurn()` now applies `damage_taken_pct` reduction from career passives

#### Capture (`src/stores/game/modules/missionModule.ts`)
- `finishCapture()` now uses `applyCaptureRateBonus()` instead of hardcoded checks for specific summoner node IDs
- Career `capture_rate_pct` and `capture_bonus_pct` are dynamically added to base capture probability

#### Fusion (`src/stores/game/modules/missionModule.ts`)
- `breedCreatures()` now applies `fusion_success_chance` as a multiplier to the fusion stat bonus
- Future fusion timer mechanics can use `fusion_timer_reduction_pct`

#### Missions (`src/stores/game/modules/missionModule.ts`)
- `addMissionWithModifiers()` now passes `playerState` to `getCareerModifiers()`
- `grantMissionXP()` now applies `xp_gain_pct` and `creature_xp_gain_pct` via `applyXPBoost()`

#### World Tick / Heartbeat (`src/stores/game/modules/missionModule.ts`)
- `startHeartbeat()` `onWorldTick` callback now executes `applyWorldTickCareerBonuses()`
- Applies passive resource regeneration boosts from career passives (`energy_regen_pct`, `nerve_regen_pct`, `happy_regen_pct`, `life_regen_pct`)
- `getCareerAggregateStats()` is called every world tick ensuring bonuses are always current

### 4. Career Tree Index Update (`src/data/careerTree/index.ts`)
- Re-exports `getCareerModifiers` from `careerTreeIntegration.ts`
- Maintains backward compatibility with existing tests
- All existing node IDs preserved for test compatibility
