# Sprint 6 – T6.4.3: Add Treasure Room Placement Logic

## Objective
Implement treasure room placement logic for dungeon generation that ensures:
- Each floor contains at least 1 treasure room
- Treasure rooms are located far from entrance (at least 60% of max distance)
- Optional secondary treasure rooms for larger floors

## What Was Found

### Existing Implementation
The treasure room placement was already partially implemented in `src/core/dungeonGenerator.ts` within `assignRoomTypes()` function (lines 151-185), but it placed treasure rooms randomly without considering distance from entrance.

### Requirements
From the task specification:
1. Each floor must contain ≥ 1 treasure room
2. Treasure room must be located far from entrance (not near the entrance)
3. Add optional secondary treasure rooms for large floors

## What Was Done

### Changes to `src/core/dungeonGenerator.ts`

#### `assignTreasureRooms()` (new function, lines 151-185)
Extracted treasure room logic into a dedicated function that:
1. Calculates room distances from entrance using `calculateRoomDistanceMap()`
2. Determines maximum distance and sets minimum threshold at 60% of max distance
3. Filters eligible rooms (non-entrance, non-boss, at least 60% of max distance away)
4. Calculates treasure count: `Math.max(3, floor(rooms.length / 10))` - scales with floor size
5. Shuffles eligible rooms and assigns first N as treasure rooms
6. Updates `treasureRoomIds` array on the graph

#### `assignOtherRoomTypes()` (new function, lines 187-194)
Extracted assignment of remaining room types (combat, trap, puzzle, rest, elite, vendor) into separate function to:
- Keep treasure room logic isolated
- Ensure proper room type distribution after treasure placement

#### Updated `assignRoomTypes()` (lines 196-212)
Modified to orchestrate the two new functions in proper order:
1. Set entrance room type
2. Set boss room type
3. Call `assignTreasureRooms()` for distance-aware placement
4. Call `assignOtherRoomTypes()` for remaining room types

### Test Coverage Added to `src/test/dungeonGenerator.test.ts`

#### New test suite "Treasure room placement logic"
- **"places treasure rooms far from entrance"**: Verifies treasure rooms are at least 60% of max distance from entrance
- **"treasure rooms are not entrance or boss rooms"**: Validates treasure rooms have distinct IDs from special rooms
- **"treasure rooms are reachable from entrance"**: Ensures all treasure rooms have valid paths
- **"guarantees at least 1 treasure room per floor"**: Minimum treasure room requirement test

## Algorithm Details

### Distance-Based Selection
Rooms must be at least 60% of the maximum distance from entrance to qualify for treasure placement. This ensures treasure rooms are meaningfully deep in the dungeon.

### Scalable Treasure Count
The maximum number of treasure rooms scales with floor size: `Math.max(3, floor(rooms.length / 10))`. Larger floors get more treasure rooms, with a floor of 3 for small floors.
