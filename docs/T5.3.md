# Task T5.3 – Implement skill inheritance: up to 3 skills from parents, highest-tier skills prioritized

## What was found

The `breedCreatures` function in `src/stores/game/modules/missionModule.ts` inherited parent skills via a simple `Set` union followed by `slice(0, 3)`:

```typescript
const parentSkills = new Set([...(c1.skills || []), ...(c2.skills || [])]);
const inheritedSkills = Array.from(parentSkills).slice(0, 3);
const finalSkills = [...inheritedSkills, ...selectedSkills.slice(0, 4 - inheritedSkills.length)];
```

This meant:
- Skills were deduplicated, but the selection order was arbitrary (based on insertion order of the Set, which follows the array order of parent1 skills then parent2 skills).
- There was no concept of skill tier or priority — a high-tier skill could be dropped in favor of a low-tier one simply because it appeared later in the Set iteration.

Additionally, `SKILL_TEMPLATES` in `src/modules/creatures/creatureFactory.ts` had no `tier` property, so the engine had no way to rank skills by power scope.

## What was done

### 1. Added `tier` to `SKILL_TEMPLATES`

Each skill entry now carries an explicit `tier` (1–5) based on its power and cost scope:

| Tier | Power range | Skills |
|------|-------------|--------|
| 1 | 10–14 | `scratch`, `stone_throw`, `vine_whip` |
| 2 | 15–17 | `wind_slash`, `water_spray`, `iron_fist`, `ice_shard` |
| 3 | 18–20 | `fire_blast`, `spark` |
| 4 | 21–24 | `holy_light`, `shadow_bolt` |

### 2. Created `src/data/fusionUtils.ts`

A pure, testable utility module exposing:

- **`getSkillPriority(skillKey)`** — looks up a skill in `SKILL_TEMPLATES` and returns `{ key, tier, power }`. Unknown skills resolve to `tier: 0, power: 0`.
- **`inheritSkills(parentASkills, parentBSkills, selectedSkills)`** — deduplicates all parent skill keys, sorts them by `tier` descending (then `power` descending as tiebreaker), takes the top 3, and pads with up to `selectedSkills` from the breeder to a maximum of 3 total inherited skills.

Algorithm:
1. Union parent skills into a deduplicated array.
2. Enrich each with `{ tier, power }` from `SKILL_TEMPLATES`.
3. Sort: `b.tier - a.tier || b.power - a.power`.
4. Slice top 3 → `inheritedSkills`.
5. Concatenate `selectedSkills.slice(0, 3 - inheritedSkills.length)`.

### 3. Updated `src/stores/game/modules/missionModule.ts`

- Imported `SKILL_TEMPLATES` and `inheritSkills`.
- Replaced the inline skill-inheritance block with a single call:
  ```typescript
  const finalSkills = inheritSkills(c1.skills || [], c2.skills || [], selectedSkills);
  ```

### 4. Updated `src/data/index.ts`

Exported `inheritSkills` from the data barrel:
```typescript
export { inheritSkills } from './fusionUtils';
```

## Behavior Summary

When two creatures fuse:
1. All unique skill keys from both parents are collected.
2. They are ranked by `tier` (highest first), with `power` as tiebreaker.
3. Top 3 are inherited.
4. If fewer than 3 slots are filled, the breeder’s `selectedSkills` fill the remainder up to a total of 3 inherited skills.
