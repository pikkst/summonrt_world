# Task T4.8 - Implement creature XP formula: `XP_Required(Level) = 50 × (1.12)^(Level − 1)` with evolution triggers

## What was found

1. **Incorrect XP formula for creatures**: The `applyCreatureXP()` function in `src/core/xpCurve.ts` was using `getXPThreshold(level)`, which implements the player XP formula (`100 × 1.15^(Level−1)`). Per the GDD, creatures require a separate, slower XP curve.
2. **Missing evolution system**: There was no creature evolution/evolution trigger system. Creatures could level up but had no mechanic to transform into stronger forms.
3. **Hardcoded combat level cap**: `combatModule.ts` had `MAX_LEVEL = 20`, which prevented creatures from reaching levels where evolution would trigger.

## What was done

### 1. Added creature-specific XP formula in `src/core/xpCurve.ts`

- **`getCreatureXPThreshold(level)`** — returns `50 × (1.12)^(Level − 1)` using `bigint` arithmetic (truncated integer division) to match the GDD creature XP curve.
- **`getCreatureCumulativeXP(level)`** — sums thresholds from 1 to `level` for total XP calculations.
- **`getCreatureXPForLevel(start, end)`** — difference between cumulative XP at two levels.
- **Updated `applyCreatureXP()`** — now uses `getCreatureXPThreshold` instead of `getXPThreshold`, so all creature XP gains (combat, missions, training, world ticks) follow the correct creature formula.
- **Updated `CreatureXPResult`** — added `evolved`, `evolutionStage`, `newClass`, and `evolutionStats` fields to surface evolution events to callers.

### 2. Implemented evolution trigger system

- **`checkEvolution(creature)`** in `xpCurve.ts` — inspects the creature's current class and `evolutionStage`, then consults `EVOLUTION_CHAINS` to see if the next evolution threshold is met.
- **Evolution data** added to `src/data/constants.ts` — defines stage transitions per class:
  - `common` → `uncommon` (Lv. 10) → `rare` (Lv. 30) → `epic` (Lv. 60)
  - `uncommon` → `rare` (Lv. 20) → `epic` (Lv. 50) → `legendary` (Lv. 80)
  - `rare` → `epic` (Lv. 40) → `legendary` (Lv. 75)
  - `epic` → `legendary` (Lv. 60)
  - `legendary` → `mythical` (Lv. 90)
- **On evolution** the creature's class is updated, stats receive a multiplicative boost (`statMultiplier`), and `evolutionStage` increments. Evolution is restricted to once per `applyCreatureXP` call to prevent chain-evolutions in a single level-up batch.
- **Added evolution fields** to `CreatureInstance` (`evolutionStage`, `evolvedFromKey`) and `CreatureTemplate` (`evolutionLevel`, `evolvesIntoKey`) in `src/types/game.ts`.

### 3. Updated combat module (`src/stores/game/modules/combatModule.ts`)

- Increased `MAX_LEVEL` from `20` to `1000` so creatures can reach evolution-eligible levels.
- Added evolution log messages (`EVOLUTION! <name> has evolved into <CLASS>!`) in `handleVictory` when `xpResult.evolved` is true.

### 4. Updated training and mission modules

- **`playerModule.ts`** (`finishActivity`) and **`missionModule.ts`** (`finishActivity` / `grantMissionXP`) now check `xpResult.evolved` and log the evolution message when a creature evolves during training or mission rewards.

### 5. Added test coverage in `src/test/xpCurve.test.ts`

- Tests for `getCreatureXPThreshold` (Lv. 1 = 50, Lv. 2 = 56, monotonic increase, large-level calculability).
- Tests for `getCreatureCumulativeXP` and `getCreatureXPForLevel`.
- Tests for `applyCreatureXP` with evolution:
  - Levels up using the new creature formula.
  - Triggers evolution at the correct class threshold.
  - Does not evolve before reaching the required level.
  - Applies stat gains on evolution.
  - Prevents double-evolution within a single XP batch.

## Summary

The creature XP system now uses the correct GDD formula `50 × (1.12)^(Level − 1)` and includes a complete evolution trigger framework. Creatures automatically evolve through rarity tiers (`common` → `uncommon` → `rare` → `epic` → `legendary` → `mythical`) at defined level thresholds, receiving class upgrades and stat multipliers. Evolution events are logged in combat, training, and mission completion flows.

All existing tests pass (151 tests), TypeScript typecheck passes, and the new creature XP formula is fully covered.
