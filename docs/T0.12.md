# T0.12 – Verify `npm run build` produces 0 errors and `dist/` is ready for deploy

## Task
Verify that `npm run build` executes without TypeScript errors and that the `dist/` output folder contains a deployable production build.

## Initial State
Running `npm run build` in `summoner-world/` failed with **48 TypeScript errors** caused by strict compiler settings (`strict: true`, `noUncheckedIndexedAccess: true`) introduced in T0.7. The build script (`tsc && vite build`) halted at the TypeScript compilation step, so no `dist/` artifacts were generated.

## Errors Found and Fixed

### `src/utils/SeededRandom.ts`
- `pick()`: Array access `arr[index]` returns `T | undefined` under `noUncheckedIndexedAccess`. Added empty-array guard and changed return type to `T | undefined`.
- `pickWeighted()`: Same issue with `items[items.length - 1]`. Added empty-array guard and non-null assertion on the fallback item.
- `shuffle()`: Destructuring swap `[a[i], a[j]] = [a[j], a[i]]` failed because indexed access is `T | undefined`. Replaced with explicit temp-variable swap.

### `src/modules/combat/combatSystem.ts`
- `generateEncounterTemplate()`: `classWeights[i]` was flagged as possibly undefined inside the weighted-random loop. Added guard `if (w === undefined) continue;`.

### `src/modules/creatures/creatureFactory.ts`
- `weightedRandomClass()`: Same loop issue with `weights[i]`. Added `if (w === undefined) continue;` guard.

### `src/core/worldGenerator.ts`
- `generateNPC()`: `rng.pick()` can return `undefined`; added `|| 'Unknown NPC'` fallback.
- `generateTile()`: Replaced inline landmark-type array indexing with a pre-captured variable to avoid unchecked-access errors.

### `src/stores/gameStore.ts`
- **Activity type (line 113)**: `GameStoreState['activity']` can be `null`, so accessing `['type']` was invalid. Changed to `NonNullable<GameStoreState['activity']>['type']`.
- **Player ID (line 208)**: `player._id || player.id` could both be `undefined`. Added `|| ''` fallback.
- **Login tile restoration**: Added guards for undefined `wid`/`coords` from split operations and length checks before indexing.
- **completeQuest()**: Added `if (!quest) return;` after `findIndex` result. Added optional chaining on `template.rewards?.element`, `?.money`, `?.exp`.
- **breedCreatures()**: Added early-return guard for `essenceStack` before inventory mutation.
- **useItem()**: Added early-return guard for `stack` before any mutations.
- **triggerEnemyTurn()**: Passed `combat.enemyHp || 0` instead of possibly undefined value.
- **handleVictory()**: Added `|| ''` and `|| 'Manifestation'` fallbacks for `combat.enemyName`.
- **descendDungeon()**: Guarded array-access for trap type selection with a temp variable and fallback.
- **rollAffinity()**: Added `as Element` casts on filtered-array accesses.
- **getAvailableSkillForCreature()**: Added `?.key || null` fallbacks on skill-array accesses.

### `src/ui/FusionPanel.tsx`
- `startFusion()`: `selectedIds[0]` and `selectedIds[1]` are possibly `undefined` at the call site. Added non-null assertions (`selectedIds[0]!`, `selectedIds[1]!`) guarded by the earlier `selectedIds.length !== 2` check.

### `src/ui/GameShell.tsx`
- **Nearby players useMemo**: Replaced direct `split(',').map(Number)` with explicit `parts` destructuring and `parseInt` + `isNaN` guard to prevent undefined `tx`/`ty`.
- **Combat panel**: Added `|| 0` fallback for `combat.enemyHp` and `combat.enemyMaxHp` in the HP display span and the HP-bar width calculation.
