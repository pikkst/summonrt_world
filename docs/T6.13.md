# Sprint 6 – T6.13: Integrate Automated Combat Engine with Timed Missions

## Objective
Integrate the automated combat engine (`resolveAutomatedCombat`) with the timed mission system so that dungeon scouting and wild world encounters are resolved automatically via missions instead of immediate combat triggers.

## What Was Found

### Existing Implementation
The automated combat engine already existed in `src/core/missionQueue.ts` as `resolveAutomatedCombat()`. It supported:
- Turn-based combat simulation (max 30 turns)
- Elemental effectiveness calculations
- XP and reward generation
- Seeded RNG for deterministic outcomes

The mission queue system had:
- `WILD_ENCOUNTER` mission type missing from `MissionType` enum
- `SCOUT_DUNGEON` mission type existed but only granted XP without combat resolution
- Wild encounters in `finishMovement()` triggered immediate `startCombat()` via `setTimeout`
- No integration between mission completion and automated combat resolution

### GDD Requirement
From the task specification:
> Integrate automated combat engine with timed missions: dungeon scouting and wild encounters resolved via `resolveAutomatedCombat()`

## What Was Done

### 1. Extended Mission Type System

### 2. Enhanced Mission Modifiers
Added `encounter_data` field to `MissionModifiers` to serialize enemy template data for timed resolution:
```typescript
export interface MissionModifiers {
  speed_multiplier?: number;
  caravan_speed_pct?: number;
  yield_bonus_pct?: number;
  tree_speed_pct?: number;
  creature_agility_mod?: number;
  resource_type?: string;
  encounter_data?: string;
  [key: string]: number | string | undefined;
}
```

### 3. Refactored Automated Combat Return Type
Changed `resolveAutomatedCombat()` return type from `MissionResult` to `AutomatedCombatOutcome` to expose both the result and the updated team states:
```typescript
export interface AutomatedCombatOutcome {
  result: MissionResult;
  teamA: CombatTeamMember[];
  teamB: CombatTeamMember[];
}
```

### 4. Converted Wild Encounters to Timed Missions
Modified `finishMovement()` in `src/stores/game/modules/missionModule.ts` to create a `WILD_ENCOUNTER` mission instead of triggering immediate combat:
```typescript
const encounterDuration = 10 + Math.floor(Math.random() * 15);
const encounterMission = get().addMissionWithModifiers({
  type: 'WILD_ENCOUNTER',
  assigned_creatures: [],
  world_layer: currentWorldId,
  duration_seconds: encounterDuration,
  extraModifiers: {
    encounter_data: JSON.stringify({
      templateKey: enemy.key,
      name: enemy.name,
      class: enemy.class,
      type: enemy.type,
      elements: enemy.elements,
      baseHealth: enemy.baseHealth,
      baseAttack: enemy.baseAttack,
      baseDefense: enemy.baseDefense,
      baseSpeed: enemy.baseSpeed,
      baseMana: enemy.baseMana,
      baseExpValue: enemy.baseExpValue,
      skills: enemy.skills,
      description: enemy.description,
      isBoss: enemy.isBoss,
    }),
  },
});
```

### 5. Integrated Combat Resolution in Mission Callbacks
Added `WILD_ENCOUNTER` handlers in both `processOfflineCatchUp()` and `startHeartbeat()` resolve callbacks. On mission completion:
1. Parse `encounter_data` from mission modifiers
2. Call `resolveAutomatedCombat()` with the player's active party vs the enemy template
3. Grant XP if the party achieves victory
4. Log the final combat message to the game log

### 6. Updated Heartbeat System
Added `WILD_ENCOUNTER` to `HeartbeatCallbacks` interface in `src/core/heartbeat.ts`:
```typescript
resolveMissionCallbacks: {
  EXPLORE_TIER_1: (mission: ActiveMission) => void;
  SCOUT_DUNGEON: (mission: ActiveMission) => void;
  ...
  DEMONLORD_ENCOUNTER: (mission: ActiveMission) => void;
  WILD_ENCOUNTER: (mission: ActiveMission) => void;
};
```

### 7. Updated Tests

Updated `src/test/heartbeat.test.ts` to include `WILD_ENCOUNTER: vi.fn()` in all `resolveMissionCallbacks` objects.

## Combat Flow (New)
```
Player moves to new tile
    ↓
Random encounter check (5% + proximity factor)
    ↓
Create WILD_ENCOUNTER mission (10-25s duration)
    ↓
Enemy template serialized into mission.modifiers.encounter_data
    ↓
Mission queued in ActiveMissionQueue
    ↓
Heartbeat tick detects mission.end_time reached
    ↓
resolveMission() calls WILD_ENCOUNTER callback
    ↓
resolveAutomatedCombat(party, [enemy])
    ↓
Victory → grantMissionXP(party, baseXP)
Defeat → no XP, log defeat message
    ↓
Mission removed from queue
```
