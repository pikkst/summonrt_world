# Task T4.9 – Implement mutation system on level-up

## What was found

1. **No mutation system existed**: The `applyCreatureXP()` function in `src/core/xpCurve.ts` handled XP accumulation, level-ups, and evolution triggers, but had no logic for mutations. The `CreatureInstance` type already had a `mutations: string[]` field (used by fusion and capture), but it was never populated by the level-up system.
2. **No mutation constants or pools**: There were no defined mutation types, max-mutation limits per class, or source pools for new skills/traits that mutations could draw from.
3. **Combat module overwrote mutation skills**: In `src/stores/game/modules/combatModule.ts`, the `handleVictory` level-up logic built `newSkills` from `c.skills` (the pre-XP skills), which would silently discard any mutation-granted skills added by `applyCreatureXP`.

## What was done

### 1. Added mutation infrastructure to `src/data/constants.ts`

- **`MUTATION_TYPES`** — `['stat_shift', 'new_skill', 'passive_trait', 'elemental_drift']`
- **`MUTATION_SKILL_POOL`** — Array of 11 creature skill keys used when a mutation grants a new skill.
- **`MUTATION_TRAIT_POOL`** — Array of 5 trait keys used when a mutation grants a passive trait.
- **`MAX_MUTATIONS_PER_CLASS`** — GDD-capped limits: Common 0, Uncommon 1, Rare 1, Epic 2, Legendary 3, Mythical 4.
- **`getClassTier(class)`** — Maps class string to 0-5 tier index.
- **`getMutationChance(tier)`** — Implements `BaseChance = 0.02 + (Tier × 0.01)`.

### 2. Implemented mutation engine in `src/core/xpCurve.ts`

- **`getMutationChanceForClass(creatureClass)`** — Public helper that computes per-class mutation probability.
- **`rollAndApplyMutation()`** — Picks a random mutation type and executes its effect:
  - `stat_shift` — Randomly boosts ATK, DEF, or SPD by +2.
  - `new_skill` — Adds a random skill from `MUTATION_SKILL_POOL` not already known.
  - `passive_trait` — Adds a random trait from `MUTATION_TRAIT_POOL` not already present.
  - `elemental_drift` — Adds a random base element not already in the creature's element list.
- **Updated `applyCreatureXP()`** — Inside the level-up loop, after evolution processing, the function checks:
  1. Has the creature reached its class mutation cap?
  2. Does `Math.random()` pass the tier-based threshold?
  3. If both pass, rolls and applies one mutation per level gained.
- **Updated `CreatureXPResult`** — Added `mutations?: string[]` to surface the list of mutation keys generated during the XP batch.
- **Updated `grantPartyXP()`** — Now returns `mutatedIds` and `mutationsById` (map of creature ID → mutation keys) so callers can log mutation events.

### 3. Fixed combat skill-learning in `combatModule.ts`

- Changed `newSkills` initialization from `[...c.skills]` to `[...xpResult.creature.skills]` so mutation-granted skills are preserved.
- Used `xpResult.creature.elements` for elemental skill filtering so `elemental_drift` mutations are correctly recognized.
- Used `xpResult.newClass || c.class` for max-skill calculation to account for evolution-then-mutation ordering.
- Added mutation log line (`🧬 <name> mutated: <labels>!`) alongside existing level-up, max-level, and evolution logs.

### 4. Added mutation logging to mission and player modules

- **`missionModule.ts` `grantMissionXP`** — Iterates over `mutatedIds` and logs new mutations using `mutationsById`.
- **`missionModule.ts` `finishActivity`** — Checks `xpResult.mutations` after creature training and logs any mutations.
- **`playerModule.ts` `finishActivity`** — Same mutation logging added for parity with mission module.

### 5. Added test coverage in `src/test/xpCurve.test.ts`

- Tests for `getMutationChanceForClass` verifying correct probability per tier (0.02–0.07).
- Tests for `getMaxMutationsForClass` verifying GDD limits.
- `applyCreatureXP` mutation tests with mocked `Math.random`:
  - Common creatures (0 max mutations) never mutate.
  - Stat-shift, new-skill, passive-trait, and elemental-drift each apply the correct state change.
  - Mutation caps are respected (Legendary capped at 3 per batch).
  - Existing `mutations` arrays are preserved and extended.

## Summary

The mutation system is now fully operational on creature level-up. Each level gained rolls against a tier-scaled probability (`0.02 + Tier × 0.01`) and, if successful, delivers one of four mutation types: stat shift, new skill, passive trait, or elemental drift. Mutations respect per-class caps from the GDD, persist on the creature, and are surfaced in the log in combat, mission rewards, and training activities. The combat skill-learning path was also corrected to stop overwriting mutation-granted skills.
