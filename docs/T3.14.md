# Task T3.14 - Verify Level 1000 Achievable Without Overflow (Use BigInt for XP Fields)

## What was found

The existing implementation had several critical issues preventing level 1000 from being achievable with correct XP calculations:

### Issue 1: `getXPThreshold` Precision Loss (xpCurve.ts:5)
The formula `Math.pow(1.15, level - 1) * 100` uses JavaScript `number` (IEEE 754 double) which has only 53 bits of mantissa (~15-17 decimal digits of precision). For level 1000, the intermediate calculation produces values around 10^62, where the unit in the last place (ULP) is approximately 10^44. This caused `Math.round()` to produce mathematically incorrect results, despite the value staying within `Number.MAX_VALUE`.

### Issue 2: Type Mismatch in Type Definitions (game.ts)
- `CreatureInstance.experience` (line 70) was typed as `number`
- `PlayerState.experience` (line 158) was typed as `number`

The cumulative XP required to reach level 1000 is approximately 2.63 Ă— 10^62, far exceeding `Number.MAX_SAFE_INTEGER` (9 Ă— 10^15). Storing XP as `number` loses precision at high levels.

### Issue 3: `applyCreatureXP` BigInt Conversion Back to Number (xpCurve.ts:112)
The function converted the bigint threshold back to `number` via `Number(getXPThreshold(newLevel))`, negating the entire benefit of using bigint.

### Issue 4: `addPlayerXP` Inline Formula (helpers.ts:53)
The function reimplemented the XP formula inline using `Math.pow(1.15, newLevel - 1)` instead of calling `getXPThreshold`, creating a duplicate source of precision loss.

### Issue 5: `tickCreatures` Precision Loss (gameEngine.ts:67-68)
Same issue as `applyCreatureXP` - converted bigint threshold back to number.

### Issue 6: Missing Verification Tests (xpCurve.test.ts)
Tests only verified that level 1000 returned a positive value, not that the value was mathematically correct or that cumulative calculations worked properly.

## What was done

### 1. Fixed `getXPThreshold` to use pure BigInt arithmetic (xpCurve.ts)
Changed from:
```typescript
const xp = Math.pow(1.15, level - 1) * 100;
return BigInt(Math.round(xp));
```

To:
```typescript
const exponent = level - 1;
const numerator = 115n ** BigInt(exponent);
const denominator = 100n ** BigInt(exponent);
return (100n * numerator) / denominator;
```

This formula computes `floor(100 * (115/100)^(level-1))` using integer-only arithmetic, ensuring exact precision for all levels including 1000.

### 2. Updated Type Definitions (game.ts)
Changed both `experience` fields from `number` to `bigint`:
- `CreatureInstance.experience: bigint`
- `PlayerState.experience: bigint`

### 3. Updated `applyCreatureXP` to use bigint internally (xpCurve.ts)
- Changed `xp` parameter to `bigint | number` for flexibility
- Used `BigInt(xp)` for conversion
- Removed `Number()` conversion when comparing against threshold
- Used `creature.experience ?? 0n` for null safety

### 4. Fixed `addPlayerXP` to use `getXPThreshold` (helpers.ts)
- Changed to use `getXPThreshold(newLevel)` directly
- Used `BigInt(Math.round(xpGained * worldModifier))` for XP conversion

### 5. Fixed `tickCreatures` in gameEngine.ts
- Changed `newExp` to `bigint` type with `BigInt(player.experience) + 1n`
- Removed `Number()` conversion when calling `getXPThreshold`
- Updated `applyCreatureXP` call to pass `1n` instead of `1`

### 6. Updated all creature instantiation sites
- `missionModule.ts`: `experience: 0n`
- `playerModule.ts`: `experience: 0n` (both initGame and useItem for boss_egg)
- `combatModule.ts`: `experience: 0n` (MAX_LEVEL reset)
- `combat.test.ts`: Updated test creature factory to use `0n`
- `careerTree.test.ts`: All player test fixtures updated to use `0n`
- `regeneration.test.ts`: Updated basePlayer fixture to use `0n`

### 7. Updated ResourcePanel.tsx
Changed XP percentage calculation to work with bigint:
```typescript
const xpPercentage = Math.min(100, Number((player.experience * BigInt(100)) / BigInt(nextLevelXP)));
```

### 8. Added comprehensive tests (xpCurve.test.ts)
- Added test verifying level 1000 threshold > 10^60
- Added test verifying level 1000 threshold grows from level 999
- Added test verifying monotonic increase of thresholds through level 100
- Added test verifying cumulative XP for level 100 and 1000
