# Sprint 6 – T6.2: Combat Phase Boss Mechanics

## Task Summary
Implement combat phase boss mechanics for bosses at HP thresholds 75%, 50%, and 25%. Each threshold triggers an elemental shift and a random environmental hazard.

## Findings

### Existing Combat System
- `src/stores/game/modules/combatModule.ts` handles the active combat loop (player turn → enemy turn).
- `src/types/game.ts` defines `CombatState`, `CreatureTemplate`, and core game types.
- No boss phase/threshold mechanics existed prior to this task.
- The `isBoss` flag and `boss`/`guardian` name check were present, but no dynamic phase transitions were implemented.
- Boss detection in `combatModule.ts` relied on enemy template `isBoss` or name matching.

### Gaps Identified
1. Bosses had no HP-based phase transitions.
2. No elemental shift mechanics tied to combat progression.
3. No environmental hazard system in active combat.

## Implementation

### Types Added (`src/types/game.ts`)
- `BossPhase` interface: `threshold`, `elementalShift`, `hazard`.
- `CreatureTemplate.bossPhases?: BossPhase[]` – data-driven phase configuration.
- `CombatState` extended with:
  - `isBoss?: boolean`
  - `bossPhasesTriggered?: boolean[]`
  - `activeBossElement?: Element`
  - `activeHazard?: string`

### Combat Logic (`src/stores/game/modules/combatModule.ts`)
- **`startCombat`** – initializes boss phase tracking when a boss is detected:
  - `bossPhasesTriggered: [false, false, false]`
  - `activeBossElement` set to the boss's first element
- **`applyBossPhaseMechanics`** – new method called after every player attack that reduces boss HP:
  - Checks HP thresholds: 75%, 50%, 25%.
  - Triggers at most one phase transition per threshold.
  - Randomly selects a new elemental affinity from the 10 base elements.
  - Randomly selects an environmental hazard description.
  - Appends phase change messages to the combat log.
  - Updates `activeBossElement` so subsequent enemy attacks use the shifted element.
- **`attackWithCreature`** – calls `applyBossPhaseMechanics()` after updating enemy HP, before triggering the enemy turn.
- **`useSkill`** – same integration as `attackWithCreature`.
- **`triggerEnemyTurn`** – updated to read the live `combat.log` from state (instead of the stale parameter) so phase messages are preserved. Elemental effectiveness calculations now use `combat.activeBossElement` when available.

### Hazard & Shift Mechanics
- **Elemental Shift** – boss randomly cycles to a new element from the base pool (`fire`, `water`, `earth`, `air`, `lightning`, `iron`, `nature`, `ice`, `light`, `darkness`). This directly affects the boss's attack elemental effectiveness against the player's creatures.
- **Environmental Hazard** – logged as flavor text at each phase transition:
  - "Poisonous fumes fill the air!"
  - "The ground cracks open with lava!"
  - "A blizzard rages across the arena!"
  - "Acidic rain pours down from above!"
  - "Scorching heat radiates from the walls!"
  - "Shadowy miasma saps the will to fight!"
