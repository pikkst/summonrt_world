# Task T4.6 - Add "All Element" (Omni) Quest-Chain Lock

## What was found

The elemental system had no mechanism for the "Omni" (All Element) unlock which is specified in the GDD as:
- Not rollable during character creation
- Only obtainable through the Convergence quest chain
- Grants +30% bonus to all elemental damage

Additionally:
1. **No "omni" element type**: The `Element` type in `src/types/game.ts` did not include 'omni'
2. **No quest chain for omni**: The `src/data/quests.ts` had no Convergence quest chain
3. **No unlock validation**: No check to verify player has all 10 base elements before granting omni
4. **No combat bonus**: No damage multiplier for omni element in combat calculations
5. **ELEMENTAL_OPPOSITIONS missing omni**: The `src/core/xpCurve.ts` didn't have omni in its opposition map

## What was done

### 1. Added `omni` to Element type (`src/types/game.ts:4-18`)
- Extended the Element union type to include `'omni'` alongside void/starlight/chaos

### 2. Updated ALL_ELEMENTS constant (`src/data/constants.ts:2`)
- Added 'omni' to the `ALL_ELEMENTS` array (quest-only elements, not rollable in `ELEMENTS`)

### 3. Created Convergence Quest Chain (`src/data/quests.ts:127-163`)
Added three quest templates:
- **convergence_part1**: "The Celestial Forge" - Acquire the Primordial Shard from Ancient Spire Floor 10
- **convergence_part2**: "Elemental Resonance" - Collect all 10 base elemental essences
- **convergence_part3**: "The Great Convergence" - Merge essences at the Convergence Altar to unlock omni

### 4. Updated quest completion logic (`src/stores/game/modules/missionModule.ts:483-525`)
- Added validation in `completeQuest()` to check if player has all 10 base elements before granting omni
- If not all elements are mastered, the quest completion logs a warning: "To unlock Omni, you must first master all 10 base elements. The convergence fails."
- Only grants omni element when all prerequisites are met

### 5. Added omni damage multiplier in combat (`src/stores/game/modules/combatModule.ts:8-21`)
- Added `getOmniDamageMultiplier()` function that returns 1.3 when player has omni element
- Added `getElementDamageMultiplier()` that combines primordial (1.2x) and omni (1.3x) multipliers
- Applied to both `attackWithCreature()` and `useSkill()` functions for elemental attacks
- Non-elemental attacks (Scratch) are unaffected by omni bonus

### 6. Updated ELEMENTAL_OPPOSITIONS (`src/core/xpCurve.ts:51-52`)
- Added `omni: []` to the opposition map since omni has no elemental weaknesses

### 7. Updated mission queue exports (`src/core/missionQueue.ts:131-177`)
- Extended `resolveAutomatedCombat` options to include `omniPct` parameter
- Added omni multiplier calculation alongside primordial multiplier for automated combat resolution

## Summary

The "All Element" (Omni) feature is now fully implemented as a quest-chain exclusive unlock:
- Available only through completing the Convergence quest chain (T4.6)
- Requires mastery of all 10 base elements (fire, water, earth, air, lightning, iron, nature, ice, light, darkness) before the final quest can grant omni
- Grants +30% elemental damage bonus in manual combat (stacking with primordial's +20%)
- Integrated into the existing elemental affinity system in `ElementalAffinity.learned` array
- Type-safe and compiles without errors under strict mode
