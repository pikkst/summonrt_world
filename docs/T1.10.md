# T1.10 – Remove old MASTER_SKILL_TREE.ts structure, replace with new JSON-based system foundation

## Findings

The old `src/data/skillsTree.ts` contained a hardcoded TypeScript object (`MASTER_SKILL_TREE`) with ~90 skill nodes defining a legacy skill tree (paths: Warrior, Soul, Scout, Merchant, Shadow, Guardian, Healer, Arcane). This structure was incompatible with the new career tree data layer introduced in Sprint 1 (T1.1–T1.9).

### Old system characteristics
- Hardcoded `SkillNode` records with `x`/`y` coordinates for UI positioning
- Used `cost` and `requirements.level` for unlock gates
- Relied on `skillPoints` and `skillsUnlocked` player state fields
- Applied immediate stat bonuses on unlock (strength, defense, speed, dexterity, hp, mana, archetype bonuses)

### New system characteristics
- JSON-driven `src/data/careerTree/nodes.json` with 19 nodes across 6 career categories (Blacksmith, Explorer, Official, Shopkeeper, Broker, Summoner) plus General hub
- `CareerNode` schema: `id`, `name`, `type` (minor/notable/keystone), `career_category`, `connections[]`, `stats{}`
- Player state uses `unspent_passive_points` and `unlocked_node_ids`
- Validation via `canUnlockNode` (Rules A/B/C from T1.6)
- Aggregate stats computed dynamically via `getAggregateStats`
- Respec via `respecAllNodes`

## Changes Made

### 1. `src/data/skillsTree.ts` – Converted to JSON-backed adapter
- Removed hardcoded `MASTER_SKILL_TREE` object (~890 lines)
- Now imports from `./careerTree/index` (`getAllNodes`)
- Provides backward-compatible `SkillNode` interface with auto-generated `x`/`y` layout coordinates
- Exports `MASTER_SKILL_TREE` and `SKILL_NODE_LIST` so existing consumers do not break immediately
- Default `cost` derived from node type (minor=1, notable=2, keystone=3)
- Default `requirements.level` set for notable (3) and keystone (10) nodes

### 2. `src/stores/gameStore.ts` – Removed unused import
- Removed `import { MASTER_SKILL_TREE } from '../data/skillsTree.ts'`
- The store composed `careerActions` which now handles tree access internally

### 3. `src/stores/game/modules/playerModule.ts` – Removed unused import
- Removed `import { MASTER_SKILL_TREE } from '../../../data/skillsTree.ts'`
- File never referenced `MASTER_SKILL_TREE`; legacy `skillPoints`/`skillsUnlocked` fields retained for save compatibility

### 4. `src/stores/game/modules/careerModule.ts` – Migrated to new career tree API
- Replaced direct `MASTER_SKILL_TREE[nodeId]` lookups with `getNodeById` and `canUnlockNode`
- `unlockSkill` now deducts from `unspent_passive_points` and appends to `unlocked_node_ids`
- Removed legacy archetype-specific stat bonus calculations (new system aggregates stats via `getAggregateStats`)
- Added `respecAllNodes` store action wrapping `respecAllNodes` from career tree index
- Uses new validation rules (points >= 1, not already unlocked, connected node unlocked)

### 5. `src/ui/SkillTreePanel.tsx` – Updated to render new career tree
- Switched imports from `skillsTree.ts` to `careerTree/index` and `careerTree/types`
- Generates node positions dynamically based on `career_category` (columns) and `type` (rows)
- Node status determined by `canUnlockNode` logic and `unspent_passive_points`
- Added Respec button wired to `respecAllNodes` action
- Tooltips updated to show `type` and `career_category` instead of legacy description/level requirements

## Verification

- `npm run typecheck` passes with 0 errors
- `npm run test` passes (7/7 careerTree tests)
- No runtime breaking changes to store composition

## Notes

- The old `MASTER_SKILL_TREE` data is no longer in source code; it is now derived from `nodes.json` at runtime.
- Legacy `skillPoints` and `skillsUnlocked` fields remain on `PlayerState` for save-file backward compatibility but are no longer written by the new career module.
- Future tasks (e.g., T1.11 career_category filter UI) can build directly on this JSON-backed foundation.
