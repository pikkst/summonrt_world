# Task T5.7 – Verify Fusion Result Rarity (Weighted Average, Capped at Legendary)

## What was found
The existing fusion system in `src/stores/game/modules/missionModule.ts` did not implement proper rarity progression. The `breedCreatures` function (fusion logic) previously used a simple hierarchical approach:

1. It selected the higher rarity class from the two parents
2. Applied arbitrary tier boosts for special events (Ancient Bloodline, Void/Stellar mutations, Aether fusion)
3. Did not follow the GDD specification for weighted average rarity calculation

## What was done

### 1. Added Rarity Calculation Functions to `src/data/fusionMatrix.ts` (lines 129-177)

**`CREATURE_CLASS_TIERS`** constant maps rarity classes to numeric tiers:
- `common: 0`, `uncommon: 1`, `rare: 2`, `epic: 3`, `legendary: 4`, `mythical: 5`

**`calculateFusionRarity(parentAClass, parentBClass)`** implements the core weighted average formula:
- Calculates `(tierA + tierB) / 2` for the weighted average
- Uses `Math.ceil()` to round up fractional averages
- Caps the result at legendary tier (4) - mythical requires special conditions

**`calculateFusionRarityWithSpecial(...)`** extends the base calculation for special events:
- `isAncient`: Raises minimum tier to epic (3)
- `isVoid` or `isStellar`: Raises tier by +1
- `isAether`: Raises tier by +1
- Without special conditions, still caps at legendary (4)
- With special conditions, can reach mythical (5)

### 2. Updated `src/stores/game/modules/missionModule.ts` (line 20)

- Imported `calculateFusionRarityWithSpecial` from fusionMatrix
- Replaced the inline rarity calculation (previous lines 717-723) with a single call to `calculateFusionRarityWithSpecial`
- The function now properly calculates rarity based on parent classes plus all special modifiers

### 3. Updated `src/data/index.ts` (line 5)

- Added export for `calculateFusionRarity`, `calculateFusionRarityWithSpecial`, and `CREATURE_CLASS_TIERS`

### 4. Added Tests in `src/test/fusion.test.ts` (lines 172-281)

- Tests for `calculateFusionRarity` with various parent combinations
- Tests for `CREATURE_CLASS_TIERS` constant
- Tests for `calculateFusionRarityWithSpecial` with special conditions
- Verified capping at legendary without special conditions
- Verified special conditions can push to epic/legendary/mythical tiers
