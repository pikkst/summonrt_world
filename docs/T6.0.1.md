# T6.0.1 – Demonlord Core Types

## Summary
Added the foundational TypeScript types `DemonlordState` and `DemonlordSkill` to `src/types/game.ts` to support the Final Boss (Demonlord) integration for Sprint 6 – Combat & Dungeon System.

## What Was Done

### 1. New Types Added to `src/types/game.ts`

#### `DemonlordSkill`
```typescript
export interface DemonlordSkill {
  key: string;
  name: string;
  description: string;
  element?: Element;
  power: number;
  cost: number;
  type: 'signature' | 'elemental_shift' | 'floor_manager' | 'aoe' | 'debuff';
  effects?: Record<string, number>;
}
```

Fields:
- `key` – Unique identifier for the skill
- `name` – Display name
- `description` – Flavor text and mechanical description
- `element?` – Optional elemental association (fire, water, etc.)
- `power` – Base damage/healing/numeric effect
- `cost` – Resource cost to use (mana, energy, etc.)
- `type` – Categorization:
  - `signature` – Unique Demonlord-only signature ability
  - `elemental_shift` – Changes the floor's elemental affinity
  - `floor_manager` – Alters dungeon layout or spawns reinforcements
  - `aoe` – Area-of-effect damage/healing
  - `debuff` – Applies negative status effects to the challenger
- `effects?` – Arbitrary key-value mechanical effects (e.g. `{ damageReduction: 20, healPercent: 10 }`)

#### `DemonlordState`
```typescript
export interface DemonlordState {
  isActive: boolean;
  currentLordPlayerId?: string;
  currentLordPlayerName?: string;
  floorMin: number;
  floorMax: number;
  influence: number;
  influenceDecayRate: number;
  activityThreshold: number;
  skills: DemonlordSkill[];
  pendingChallenges: Array<{
    challengerId: string;
    challengerName: string;
    issuedAt: number;
  }>;
  activeChallenge?: {
    challengerId: string;
    challengerName: string;
    acceptedAt: number;
  };
  defeatedAt?: number;
}
```

Fields:
- `isActive` – Whether a Demonlord currently rules the dungeon floors
- `currentLordPlayerId?` – Player ID of the reigning Demonlord (when a player holds the title)
- `currentLordPlayerName?` – Display name of the reigning Demonlord
- `floorMin` – Lowest floor the Demonlord can influence (50)
- `floorMax` – Highest floor the Demonlord rules (100+)
- `influence` – 0–100 scale representing AI/player control over the floor
- `influenceDecayRate` – How quickly influence decays when activity drops
- `activityThreshold` – Player activity level below which Demonlord AI gains more control
- `skills` – The Demonlord's skill set (array of `DemonlordSkill`)
- `pendingChallenges` – Queue of challenge requests from players waiting for acceptance
- `activeChallenge?` – Currently ongoing accepted challenge
- `defeated?` – Timestamp of the last defeat (used for cooldown/respawn)

### 2. Store Type Exports Updated
`src/stores/game/types.ts` was updated to import and re-export `DemonlordState` and `DemonlordSkill` so they are available throughout the Zustand store layer and all module files.

## Design Rationale

- **Skill typing** follows the existing `Skill` interface pattern used by `CreatureTemplate` but adds a `type` discriminator to distinguish Between role-based Demonlord abilities.
- **Floor range** is configurable via `floorMin`/`floorMax` to support future dungeon system changes (T6.0.5).
- **Influence model** bridges the AI activity-based influence requirement from the GDD: when fewer players are on the floor and activity is low, `influence` grows and the Demonlord AI behaves more aggressively.
- **Challenge system** uses a simple FIFO queue (`pendingChallenges`) with a single `activeChallenge` slot, matching the "issue challenge → Demonlord must accept" flow described in the task.
