# Task T3.9 – Add level-up visual feedback (log + overlay) in mission completion callback

## What was found

The mission completion system already granted XP to creatures via `grantMissionXP` (added in T3.7), but level-up events produced only a minimal success log entry from `addPlayerXP` for the player character. Creature level-ups during mission resolution had **no dedicated log entry and no visual feedback** in the UI.

Key gaps:
- `applyCreatureXP` in `src/core/xpCurve.ts` correctly tracked `leveledUp` and `newLevel`, but `grantPartyXP` only returned `leveledUpIds` — the caller `grantMissionXP` ignored this data.
- `GameShell.tsx` had progress overlays for exploration and capture, but **no level-up notification overlay**.
- Players had to check the creature panel manually to notice that a creature had leveled up after a mission.

## What was done

### 1. Extended `GameStoreState` with `levelUpNotifications`

Added a new state field to `src/stores/game/types.ts`:

```typescript
levelUpNotifications: Array<{ creatureName: string; newLevel: number }>;
```

And two new actions to `GameActions`:
- `showLevelUpNotification(notifications)` — sets the notification array.
- `clearLevelUpNotifications()` — resets the array to empty.

### 2. Implemented notification actions in `playerModule.ts`

Added the two new action implementations in `src/stores/game/modules/playerModule.ts`:

```typescript
showLevelUpNotification: (notifications) => set({ levelUpNotifications: notifications }),
clearLevelUpNotifications: () => set({ levelUpNotifications: [] }),
```

### 3. Updated `grantMissionXP` to surface creature level-ups

Modified `src/stores/game/modules/missionModule.ts` so `grantMissionXP` now:

- Destructures `leveledUpIds` from `grantPartyXP`.
- For each creature that leveled up, appends a log entry: `⚡ <name> reached Level <level>!` (type: `success`).
- Calls `showLevelUpNotification` with the list of leveled-up creatures.
- Always returns `{ leveledUpIds }` to match the updated `GameActions` interface.

### 4. Added fullscreen level-up overlay to `GameShell.tsx`

Added a fixed-position overlay in `src/ui/GameShell.tsx` that renders when `levelUpNotifications` is non-empty:

- **Z-index 200** — renders above all modals and panels.
- **Centered card** with `🎉` icon, "Level Up!" heading, and a list of each creature name + new level.
- **Auto-dismiss after 4 seconds** via a `useEffect` + `setTimeout` that calls `clearLevelUpNotifications`.
- **Pointer-events-none** on the outer wrapper so the overlay does not block UI interaction.

### 5. Initialized state in `gameStore.ts`

Added `levelUpNotifications: []` to the `initialGameState` object so the field is present from store creation.

## Files Modified

- `summoner-world/src/stores/game/types.ts` — Added `levelUpNotifications` to state and actions.
- `summoner-world/src/stores/gameStore.ts` — Initialized `levelUpNotifications: []`.
- `summoner-world/src/stores/game/modules/playerModule.ts` — Added `showLevelUpNotification` and `clearLevelUpNotifications` actions.
- `summoner-world/src/stores/game/modules/missionModule.ts` — Updated `grantMissionXP` to log creature level-ups and trigger the overlay.
- `summoner-world/src/ui/GameShell.tsx` — Added level-up visual overlay with auto-dismiss.

## Behavioral Changes

| Aspect | Before | After |
|--------|--------|-------|
| Creature level-up on mission | Silent state change | Log entry `⚡ <name> reached Level X!` |
| Visual feedback | None | Fullscreen overlay listing all creatures that leveled up |
| Overlay duration | N/A | 4 seconds, auto-dismiss |
| Overlay blocking | N/A | `pointer-events-none` — does not block clicks |
