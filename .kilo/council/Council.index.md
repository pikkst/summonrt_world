# Council Index

Use this file before opening individual council role files.

## Executive Council

- `executive/ChiefArchitect.md` - architecture boundaries, system ownership, ADR-level decisions.
- `executive/TechnicalDirector.md` - technical feasibility, stack direction, implementation constraints.
- `executive/GameplayDirector.md` - core gameplay direction and Design Constitution alignment.
- `executive/ReleaseManager.md` - release readiness and PR delivery quality.

## Design Council

- `design/CreatureDesigner.md` - creature systems, contracts, fusion, genetics, behavior.
- `design/DungeonDesigner.md` - dungeon towers, floors, traversal, rewards, bosses.
- `design/WorldDesigner.md` - world generation, world memory, ecology, settlements.
- `design/EconomyDesigner.md` - currency, rewards, trade, market balance.
- `design/UXDesigner.md` - UI flow, decision over clicking, report UX.
- `design/NarrativeDesigner.md` - lore, world story, NPC memory, player identity.

## Engineering Council

- `engineering/LeadProgrammer.md` - implementation structure and code maintainability.
- `engineering/QAEngineer.md` - tests, validation, regression coverage.
- `engineering/PerformanceEngineer.md` - scale, simulation cost, rendering cost.
- `engineering/SecurityEngineer.md` - data safety, validation, future online risks.
- `engineering/DocumentationEngineer.md` - docs, tasks, templates, architecture notes.

## Special Roles

- `AIHistorian.md` - ADRs, lessons learned, architecture memory.
- `PlayerAdvocate.md` - player value and experience quality.
- `SystemIntegrator.md` - cross-system integration and dependency fit.

## Token Rule

Open this index first.

Then open only the role files needed for the current task.
