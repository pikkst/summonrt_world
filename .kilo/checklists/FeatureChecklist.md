# Feature Checklist

Use this checklist for new gameplay, UI, technical, or framework features.

## Scope

- [ ] The feature matches the requested task.
- [ ] No unrelated refactor is included.
- [ ] Existing implementation was searched before creating new files.
- [ ] The feature has a clear owner module.

## Player Value

- [ ] The feature gives the player a meaningful choice or clear value.
- [ ] Decision over clicking was considered.
- [ ] Time-as-gameplay was considered if the feature is duration-based.
- [ ] Reports are considered when outcomes should be summarized.

## Architecture

- [ ] Relevant agent role was selected.
- [ ] Relevant council roles were selected only if needed.
- [ ] Relevant patterns were used or explicitly ruled out.
- [ ] Event needs were considered.
- [ ] Save/load impact was considered.

## Validation

- [ ] Tests were added or updated when needed.
- [ ] Documentation impact was handled.
- [ ] Known limitations are listed in PR notes.
- [ ] Validation commands were run or honestly reported.
