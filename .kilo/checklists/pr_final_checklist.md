# Pull Request Final Checklist

Use this checklist immediately before opening or updating a Pull Request.

## Branch

- [ ] Work is on a dedicated branch.
- [ ] Branch was created from `master`.
- [ ] Branch name reflects task or purpose.
- [ ] Work is not committed directly to `master`.

## Scope

- [ ] PR contains one logical task or purpose.
- [ ] No unrelated files are changed.
- [ ] No unrelated refactors are included.
- [ ] New files are placed in the correct folder.

## Quality

- [ ] TypeScript quality is preserved if code changed.
- [ ] Tests are added or updated if code changed.
- [ ] Documentation is updated if behavior, architecture, or task status changed.
- [ ] Validation results are listed honestly.

## PR Body

- [ ] Summary explains what changed.
- [ ] Summary explains why it changed.
- [ ] Validation section is present.
- [ ] Documentation section is present.
- [ ] Notes section includes known limitations or says none.

## Final Decision

- [ ] PR is ready for review.
- [ ] If not ready, PR is marked draft or remaining work is clearly stated.
