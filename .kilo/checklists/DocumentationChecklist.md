# Documentation Checklist

Use this checklist for documentation updates and docs impact review.

## Accuracy

- [ ] Documentation matches actual project state.
- [ ] Planned work is not described as completed.
- [ ] Known limitations are stated.
- [ ] Terminology matches existing project language.

## Scope

- [ ] Only relevant sections were changed.
- [ ] No unrelated rewrite is included.
- [ ] Task status changes are supported by implementation evidence.
- [ ] Links or references are updated when needed.

## AI Framework Docs

- [ ] `.kilo` docs are updated when AI workflow changes.
- [ ] Index files are updated when new documents are added.
- [ ] Token usage impact is considered for large docs.

## PR Notes

- [ ] PR explains documentation-only status if no runtime code changed.
- [ ] Validation section honestly states what was not run.
