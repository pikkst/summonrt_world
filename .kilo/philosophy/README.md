# Kilo Philosophy

This folder contains design philosophy guidance for AI coding agents working on SummonerWorld.

These files explain why systems should be designed in a specific direction. They complement rules, prompts, checklists, examples, context maps, and agent roles.

## Required Source

Read `DESIGN_CONSTITUTION.md` before reading these files.

## Philosophy Index

- `00_Core_Philosophy.md` - core loop and identity.
- `01_Player_First.md` - player journey as center.
- `02_Commander_RPG.md` - player commands, systems execute.
- `03_Decision_Over_Clicking.md` - prefer decisions over repetition.
- `04_Time_As_Gameplay.md` - timers and duration as meaningful gameplay.
- `05_Action_System.md` - Actions as universal gameplay units.
- `06_Report_System.md` - outcomes return as Reports.
- `07_World_Memory.md` - world remembers important consequences.
- `08_Offline_First.md` - offline gameplay remains core.
- `09_MMO_Ready.md` - future online compatibility.
- `10_Event_Driven.md` - events as cross-system boundaries.

## Rule

If a feature conflicts with these philosophy files, stop and re-check `DESIGN_CONSTITUTION.md` before implementing.
