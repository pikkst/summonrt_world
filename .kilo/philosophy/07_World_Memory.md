# 07 - World Memory

## Principle

The world should remember important player actions and simulation outcomes.

World Memory turns events into long-term consequences.

## What Should Be Remembered

Examples:

- bosses defeated
- dungeon floors cleared
- quests completed
- settlements helped or harmed
- creatures protected or overhunted
- resources depleted
- buildings constructed
- trade routes changed
- reputation gained or lost
- Demonlord reign changes

## Gameplay Impact

Memory should affect future gameplay through:

- NPC dialogue
- quest availability
- prices
- world events
- settlement growth
- creature populations
- dungeon difficulty
- player titles
- reputation

## AI Rule

If an event is important enough to change the player's story, consider whether it should update memory.
