# 02 - Commander RPG

## Principle

The player is a Summoner commander.

The player gives direction.

Creatures, systems, workers, NPCs, and world processes execute that direction.

## Gameplay Meaning

The player should spend most time deciding:

- which Action to start
- which creature or team to assign
- how much risk to accept
- which strategy to choose
- which reward path matters most
- when to wait and when to intervene

## Execution Meaning

Execution can happen through:

- timers
- simulations
- queues
- automated combat
- reports
- event logs
- world memory updates

## AI Rule

Do not default to manual micro-click gameplay.

First ask whether the player should command the action and receive a report instead.
