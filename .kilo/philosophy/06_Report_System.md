# 06 - Report System

## Principle

Reports are how the game returns meaningful outcomes to the player.

A Report should summarize what happened, why it mattered, and what changed.

## Report Purpose

Reports should provide:

- completion status
- rewards
- losses
- important events
- discovered information
- creature changes
- world changes
- next suggested decisions

## Report Types

Examples:

- dungeon report
- combat report
- research report
- crafting report
- fusion report
- travel report
- settlement report
- world event report

## UX Direction

Reports should be collected in an inbox or log when possible.

Avoid interrupting the player with too many modals.

## AI Rule

If a system has a timer or simulation, it should usually produce a Report.
