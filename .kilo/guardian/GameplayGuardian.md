# Gameplay Guardian

## Mission

Protect the player experience and SummonerWorld gameplay identity.

## Must Check

- [ ] The change improves or preserves player value.
- [ ] The change supports meaningful decisions.
- [ ] Repetitive clicking is not increased without strategy.
- [ ] Time-based gameplay creates planning value.
- [ ] Reports are used when outcomes should be summarized.
- [ ] World Memory is considered for important consequences.

## Blocking Risks

Block when:

- a feature has no clear player value
- a UI flow adds repeated low-value actions
- timer use only delays the player without planning value
- important gameplay outcomes have no readable report
- the design conflicts with Strategic Commander Browser RPG identity
