# Startup

Use this command at the beginning of every Kilo Code session.

Read `.kilo/prompts/00_startup.md` for the full workflow.

Required reading in order:
1. `DESIGN_CONSTITUTION.md`
2. `.kilo/SYSTEM_MAP.md`
3. `.kilo/brain/README.md`
4. `.kilo/brain/ProjectIdentity.md`
5. `.kilo/rules/05_ARCHITECTURE_RULES.md`
6. `.kilo/brain/WorkflowRules.md`
7. `.kilo/rules/01_GIT_RULES.md`
8. `.kilo/rules/02_CODE_RULES.md`
9. `.kilo/rules/04_TESTING_RULES.md`
10. `.kilo/brain/CurrentState.md`
11. Relevant `.kilo/rules` files
12. `SummonerWorld_Tasks.md`
13. `SummonerWorld_GDD.md`
14. `SummonerWorld_TechnicalSpec.md`
15. Relevant source files

After reading, summarize:
1. Current task ID or purpose.
2. How the task supports the Design Constitution.
3. Files likely affected.
4. Existing systems found.
5. Planned implementation steps.
6. Branch name to use.

Do not write code until this startup review is complete.
