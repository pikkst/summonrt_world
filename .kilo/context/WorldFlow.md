# World Flow

## Generation Flow

```text
Seed
  -> Biomes
  -> Rivers
  -> Mountains
  -> Settlements
  -> Roads
  -> Dungeons
  -> Creatures
  -> Resources
```

## Progression Flow

```text
World 1
  -> World Boss Defeated
  -> World 2
  -> ...
  -> World 100
  -> Demonlord End Game
```

## World Memory Flow

```text
Player Action
  -> Domain Event
  -> World Memory Update
  -> Future Gameplay Change
```

## World Memory Examples

- dungeon cleared
- boss defeated
- resource depleted
- settlement grown
- creature protected
- trade activity increased
- reputation changed

## Rule

Worlds should not be static backgrounds.

World state should eventually react to meaningful player actions.
