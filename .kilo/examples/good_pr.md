# Good Pull Request Example

## Purpose

Pull Requests should be easy to review and honest about validation.

## Example Structure

Title:

Add World 10 dungeon traversal test

Summary:

- Adds integration coverage for World 10 dungeon tower traversal.
- Verifies floor reachability, entrance-to-boss paths, and final boss access.
- Completes task T6.12.

Validation:

- typecheck: passed
- lint: passed
- test: passed
- build: passed

Documentation:

- task list updated
- no GDD change needed

Notes:

- no runtime gameplay logic changed

## Why This Is Good

- The task is clear.
- The validation status is explicit.
- The scope is small and reviewable.
