# Good Commit Message Example

## Purpose

Commit messages should explain the change clearly.

## Good Examples

```text
Implement T6.12 world 10 dungeon traversal test
Fix boss floor reachability validation
Add Kilo workflow prompts
Refactor dungeon pathfinding helpers
Update Player Core documentation
```

## Poor Examples

```text
fix
stuff
updates
more changes
work
```

## Recommended Pattern

Use a verb and a clear object:

```text
Add [thing]
Fix [problem]
Implement [task]
Refactor [system]
Update [documentation]
```

## Why This Is Good

- Git history becomes easier to read.
- PR review is easier.
- Future AI agents can understand project history faster.
